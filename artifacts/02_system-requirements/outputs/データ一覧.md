## 2. データ詳細定義（データID単位：任意）

### 2.1 データID：DATA-001（データ名：社内申請ルール）
- 種別：ファイル
- 概要：申請種別・申請書・申請先・承認ルールを定義した社内規定ドキュメント
- 形式（拡張子/スキーマ）：テキストドキュメント（PDF/Word等）
- 保管場所：ファイルストレージ
- 更新頻度：規程改訂時
- 参照方法（API/検索/ダウンロード等）：ファイル参照（CAP-INT-001）
- データ分類・機密区分：D-1（社内） / 社内
- マスキング要件：不要
- 保持期間：要件上未定義
- 品質要件（欠損率/更新遅延/正確性など）：最新版が常にファイルストレージに格納されていること
- 監査要件：参照ログを記録すること

#### スキーマ（要件定義レベル：代表項目のみ）
| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|
| 1 | 申請種別名 | string | ○ | 申請の種別名称 | 「交通費精算申請」 |
| 2 | 申請条件 | string | ○ | 申請種別が必要となる条件 | 「社外出張が発生する場合」 |
| 3 | 対応申請書名 | string | ○ | 使用する申請書名 | 「交通費精算申請書」 |
| 4 | 申請先 | string | ○ | 提出先部門・システム | 「総務部」 |
| 5 | 承認権限・上限 | object | — | 上長承認が必要な条件・金額上限 | 交通費：10,000円超 |

## 3. AIエージェント向け追加項目（ファイル参照系データ）

### DATA-001（社内申請ルール）

| 項目 | 内容 |
|---|---|
| 参照範囲 | 全社申請ルール（交通費精算申請・経費精算申請に関する条項） |
| 参照方針 | ファイルとして管理するナレッジを直接参照（RAG/ベクトルDB不使用） |
| メタデータ | 規程名、版、発効日、申請種別カテゴリ |
| 根拠提示 | ルールID（JUDGE-01等）・ルール概要をLv2形式で提示 |
| 更新検知 | 版管理（管理部門による更新時にファイルを差し替え） |

### DATA-004（申請先情報）

| 項目 | 内容 |
|---|---|
| 参照範囲 | 申請種別ごとの申請先（部門・担当者・システム名） |
| 参照方針 | ファイルとして管理するナレッジを直接参照 |
| メタデータ | 申請種別名、申請先部門、更新日 |
| 根拠提示 | 申請先名称をそのまま提示（Lv2）。ルールID：BRL-02 |
| 更新検知 | 組織変更時に管理部門がファイルを差し替え |

### DATA-005（申請書必須項目リスト）

| 項目 | 内容 |
|---|---|
| 参照範囲 | 申請種別ごとの必須項目リスト |
| 参照方針 | ファイルとして管理するナレッジを直接参照 |
| メタデータ | 申請種別名、テンプレート版 |
| 根拠提示 | 不要（内部処理での充足確認のみ） |
| 更新検知 | テンプレート改訂時に管理部門がファイルを差し替え |

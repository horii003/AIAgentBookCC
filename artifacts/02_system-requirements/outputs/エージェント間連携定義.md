## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（申請種別確定・申請書生成）
- 失敗時方針：人へエスカレーション（CF-006に移行）
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが**並行/分担**して成果を統合
- 本システムの適用：なし（エージェントは直列委譲構成のため協調連携なし）

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを**丸投げ**して結果を受け取る
- 本システムの適用：AG-001→AG-002/AG-003（BIZ-02委譲）

### 3.3 イベント（Event-driven）
- 目的：状態変化（例：申請書生成完了）を発火し、購読側が処理
- 本システムの適用：なし（非同期イベントバス不使用）

## 4. 連携インタフェース定義（最小セット）

### 4.1 連携ID：LNK-001（AG-001 → AG-002）

#### 4.1.1 トリガー条件
- 発火条件：AG-001が申請種別=交通費精算申請を確定した時（JD-01またはJD-02経由）
- 事前条件（Pre-condition）：D-003の申請種別が「交通費精算申請」に確定していること、申請者名（D-016）が収集済みであること
- 事後条件（Post-condition）：AG-002が起動して交通費精算申請フロー（CF-003）を開始すること

#### 4.1.2 入力スキーマ（構造化推奨）
```json
{
  "request_id": "string",
  "task": "交通費精算申請フロー開始",
  "context": {
    "application_type": "交通費精算申請",
    "applicant_name": "string",
    "user_input_text": "string"
  },
  "constraints": {
    "autonomy_level": 3
  },
  "approval_required": false
}
```

#### 4.1.3 出力スキーマ（構造化推奨）
```json
{
  "request_id": "string",
  "status": "completed | error",
  "output": {
    "draft_application_path": "string",
    "application_type": "交通費精算申請"
  },
  "error": null
}
```

#### 4.1.4 エラー時の動作
- AG-002が起動に失敗した場合：AG-001がCF-006（エラー・エスカレーション案内）に移行する
- AG-002が処理中にエラーとなった場合：AG-002がCF-006に移行して社員に案内する

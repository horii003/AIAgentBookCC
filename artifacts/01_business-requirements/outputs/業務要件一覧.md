## ターゲットユーザー
社内の申請手続きを行う一般社員（申請ルールや申請方法が不明な利用者を含む）

## 要求事項
- RQ-01: 社員が申請内容を入力すると、必要な申請種別・申請書・申請先を提示する
- RQ-02: 社内の申請ルールを参照して正しい申請種別を判断する
- RQ-03: 不足情報を対話で確認しながら申請書を自動作成する
- RQ-04: 申請内容をチェックして申請ミスや差し戻しを減らす
- RQ-05: 申請ルールや申請方法が分からない利用者でも正しく申請できるようにする

## 業務要件

| ID    | 業務要件 | 関連要求  | AI関与 | 優先度 |
| ----- | -------- | --------- | ------ | ------ |
| BR-01 | 社員が入力した申請内容をもとに必要な申請種別を判断する | RQ-01, RQ-02 | 実行 | High |
| BR-02 | 申請に使用すべき申請書と申請先を社員に提示する | RQ-01 | 実行 | High |
| BR-03 | 社内申請ルールを参照して申請種別の判断根拠を提供する | RQ-02 | 実行 | High |
| BR-04 | 申請書作成に必要な不足情報を対話で確認する | RQ-03 | 実行 | High |
| BR-05 | 収集した情報をもとに申請書を自動作成する | RQ-03 | 実行 | High |
| BR-06 | 申請内容の不備・ミスを事前にチェックする | RQ-04 | 実行 | High |
| BR-07 | チェック結果をもとに修正点を社員に提示する | RQ-04 | 提案 | High |
| BR-08 | 申請ルールを知らない社員でも正しく申請できるよう案内する | RQ-05 | 提案 | High |

## 業務種別ごとの業務ルール

### 申請種別判断の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| JUDGE-01 | 申請種別の限定 | 交通費精算申請・経費精算申請の2種別を判断対象とする | High |
| JUDGE-02 | 申請種別の判断不能ケース | 申請内容から交通費精算申請・経費精算申請のいずれかを判断できない場合は、ユーザーに選択肢を提示して確認する | High |
| JUDGE-03 | 申請先の特定 | 申請種別ごとに申請先（部門・担当者・システム）を特定して提示する | High |
| JUDGE-04 | 申請ルール参照 | 判断にあたって社内申請ルールドキュメントを参照する | High |

### 対話・情報収集の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| DIALOG-01 | 不足情報の対話確認 | 申請書作成に必要な項目（金額・日時・目的地等）が不足している場合は、追加質問で情報を収集する | High |
| DIALOG-02 | 一問一答形式 | 複数の不足項目がある場合は、一度に複数質問せず順序立てて確認する | Medium |
| DIALOG-03 | 入力内容の確認 | 収集した情報を申請書作成前に社員に確認提示する | Medium |

### 申請書自動作成の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| FORM-01 | 申請書テンプレート適用 | 申請種別に対応した申請書テンプレートを使用して申請書を生成する | High |
| FORM-02 | 収集情報の自動入力 | 対話で収集した情報を申請書の対応フィールドに自動入力する | High |
| FORM-03 | 必須項目の充足確認 | 申請書の必須項目がすべて入力されていることを確認してから生成する | High |

### 申請内容チェックの業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| CHECK-01 | 必須項目チェック | 申請書の必須項目が未入力でないかを確認する | High |
| CHECK-02 | 申請ルール適合チェック | 申請内容が社内申請ルールに適合しているか確認する | High |
| CHECK-03 | 金額・上限チェック | 申請金額が承認権限・上限規定に適合しているか確認する（交通費：10,000円超、経費：5,000円超は上長承認が必要） | High |
| CHECK-04 | チェック結果の提示 | チェックで発見した不備・修正点を具体的に社員に提示する | High |

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRANS-01 | 移動情報の一区間ごと収集 | 移動日・出発地・目的地・交通手段・費用を一区間ずつ収集する | High |
| TRANS-02 | 対応交通手段の制限 | 対応交通手段は電車・バス・タクシー・飛行機の4種類に限定する | High |
| TRANS-03 | 交通費の自動計算 | 電車は経路テーブル検索、バス・タクシー・飛行機は固定運賃で交通費を自動計算する | High |
| TRANS-04 | 申請期限の確認 | 申請は経費発生日から90日以内であること | High |
| TRANS-05 | 高額時の上長承認 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | High |
| TRANS-06 | 駅名の正規化 | 入力された駅名を正規化する（例：「渋谷駅」→「渋谷」） | Medium |
| TRANS-07 | 業務目的の必須記載 | すべての申請に業務目的の記載が必要 | High |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXPENSE-01 | 領収書情報の自動抽出 | 領収書画像から店舗名・金額・日付・品目を自動抽出する | High |
| EXPENSE-02 | 経費区分の自動判断 | 品目から経費区分（事務用品費・宿泊費・資格精算費・その他経費）を自動判断する | High |
| EXPENSE-03 | 申請期限の確認 | 申請は経費発生日から90日以内であること | High |
| EXPENSE-04 | 高額時の上長承認 | 経費が5,000円を超える場合は事前に上長の承認が必要 | High |
| EXPENSE-05 | 業務目的の必須記載 | すべての申請に業務目的の記載が必要 | High |

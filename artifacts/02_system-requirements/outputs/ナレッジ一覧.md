## 2. ナレッジ詳細定義（ナレッジID単位）

### 2.1 ナレッジID：KN-001
### ナレッジ名：社内申請ルール（申請種別判断）

---

#### 2.1.1 ナレッジ概要
- 申請内容（自由文）から申請種別（交通費精算申請・経費精算申請）を特定するための判断ルール
- 申請フローの起点として必須のナレッジ。エージェントは必ずこのナレッジに従って判断する

---

#### 2.1.2 ナレッジ種別・特性

| 項目 | 内容 |
|---|---|
| 種別 | 規程・ポリシー |
| 判断の性質 | 決定的（申請種別を確定させる） |
| 判断の粒度 | 厳密 |
| 競合時の優先度 | 高 |

---

#### 2.1.3 利用範囲・制約

| 項目 | 内容 |
|---|---|
| 利用可能エージェント | AG-001 |
| 利用可能自律度 | 最大 Lv5 |
| 利用禁止ケース | なし |
| 人判断が必要な条件 | 申請種別が特定できない場合（JD-02 NG）はユーザーに2択を提示（CF-002） |

---

#### 2.1.4 判断拘束力
- 本ナレッジの拘束力：**強**（社内規程に基づくルールであり、エージェントは必ず従う）

---

#### 2.1.5 入力・参照条件

| No | 条件項目 | 内容 |
|---:|---|---|
| 1 | 前提条件 | 社内申請ルールがファイルストレージに配置されていること（RAGは使用しない） |
| 2 | 判断条件 | ユーザーの申請内容（自由文）が入力されていること |
| 3 | 参照トリガー | CF-001 Step 4（申請種別判定）で参照 |

---

#### 2.1.6 ナレッジ構造定義

**a. 文書型（ファイル参照）**
- 保管場所：DATA-001（ファイルストレージ/ナレッジ領域）
- 参照方式：エージェントがファイルを読み込みプロンプトに埋め込み参照。RAGは使用しない
- メタデータ：ルールID, 申請種別, 版, 発効日

**b. ルール型（主要判断条件）**
```text
IF 申請内容から「交通費精算申請」と特定できる
THEN 申請種別 = 交通費精算申請（JD-01 OK）
ELSE IF 申請内容から「経費精算申請」と特定できる
THEN 申請種別 = 経費精算申請（JD-01 OK）
ELSE 申請種別が特定できない → 2択をユーザーに提示（JD-02 NG → CF-002）
```

### 2.2 ナレッジID：KN-004
### ナレッジ名：申請期限ルール（交通費・経費共通）

---

#### 2.2.1 ナレッジ概要
- 申請書は経費発生日から3ヶ月以内に提出しなければならないことを定義したルール
- 超過時は申請不可として通知する。交通費精算申請・経費精算申請の両方に適用

---

#### 2.2.2 ナレッジ種別・特性

| 項目 | 内容 |
|---|---|
| 種別 | 規程・ポリシー |
| 判断の性質 | 決定的（申請可否を確定させる） |
| 判断の粒度 | 厳密（日付計算） |
| 競合時の優先度 | 高 |

---

#### 2.2.3 利用範囲・制約

| 項目 | 内容 |
|---|---|
| 利用可能エージェント | AG-002（交通費）, AG-003（経費） |
| 利用可能自律度 | 最大 Lv5 |
| 利用禁止ケース | なし |
| 人判断が必要な条件 | なし（3ヶ月超の場合は申請不可を自動通知） |

---

#### 2.2.4 判断拘束力
- 本ナレッジの拘束力：**強**（社内規程に基づく期限ルール。例外なし）

---

#### 2.2.5 入力・参照条件

| No | 条件項目 | 内容 |
|---:|---|---|
| 1 | 前提条件 | 経費発生日が収集済みであること |
| 2 | 判断条件 | 申請日と経費発生日の差が3ヶ月以内かどうか |
| 3 | 参照トリガー | CF-005またはCF-006でのルールチェック時（申請チェック） |

---

#### 2.2.6 ナレッジ構造定義

**b. ルール型（主要判断条件）**
```text
IF (申請日 - 経費発生日) <= 3ヶ月
THEN 申請期限チェック = OK（JD-09またはJD-13）
ELSE 申請期限チェック = NG → 申請不可を通知（JD-10またはJD-14、CF-008へ）
```

---

### 2.3 ナレッジID：KN-005
### ナレッジ名：上長承認要否ルール（交通費・経費）

---

#### 2.3.1 ナレッジ概要
- 交通費精算申請は合計10,000円超、経費精算申請は5,000円超の場合、事前に上長の承認が必要なことを定義したルール
- 上長承認の取得はユーザーが行う。エージェントは確認メッセージを提示するのみ

---

#### 2.3.2 ナレッジ種別・特性

| 項目 | 内容 |
|---|---|
| 種別 | 規程・ポリシー |
| 判断の性質 | 決定的（上長承認要否を確定させる） |
| 判断の粒度 | 厳密（金額閾値との比較） |
| 競合時の優先度 | 高 |

---

#### 2.3.3 利用範囲・制約

| 項目 | 内容 |
|---|---|
| 利用可能エージェント | AG-002（交通費）, AG-003（経費） |
| 利用可能自律度 | 最大 Lv5 |
| 利用禁止ケース | なし |
| 人判断が必要な条件 | 上長承認要の場合、ユーザーが上長承認済みであることを確認する（CF-009） |

---

#### 2.3.4 判断拘束力
- 本ナレッジの拘束力：**強**（社内規程に基づく承認ルール）

---

#### 2.3.5 入力・参照条件

| No | 条件項目 | 内容 |
|---:|---|---|
| 1 | 前提条件 | 申請金額（交通費合計または経費金額）が確定していること |
| 2 | 判断条件 | 金額が閾値（交通費：10,000円 / 経費：5,000円）を超えているか |
| 3 | 参照トリガー | CF-005またはCF-006でのルールチェック時 |

---

#### 2.3.6 ナレッジ構造定義

**b. ルール型（主要判断条件）**
```text
交通費精算申請の場合：
IF 交通費合計 <= 10,000円 THEN 上長承認不要（JD-11）
ELSE 上長承認必要 → 確認メッセージ提示（JD-12、CF-009へ）

経費精算申請の場合：
IF 経費金額 <= 5,000円 THEN 上長承認不要（JD-15）
ELSE 上長承認必要 → 確認メッセージ提示（JD-16、CF-009へ）
```

## 3. ナレッジ運用上の重要な整理軸（設計標準）

### ① 「ナレッジ＝RAG」ではない
- エージェントの判断に使うナレッジのみを列挙する
- 単なる参考資料・補助情報は含めない

### ② 判断拘束力を必ず定義
- 強（必ず従う）／中（原則従うが例外あり）／弱（参考情報）
- KN-007（差し戻しリスク判定基準）は詳細未定義のため弱

### ③ 自律度と必ずペアで管理
- KN-009（申請書提出禁止ルール）は自律度Lv0（禁止）— AIは提出操作を行わない
- KN-007（差し戻しリスク）は自律度Lv2（提案のみ）— 実行権限なし
- KN-001〜KN-006はLv5まで使用可能（自動実行・判定を許可）

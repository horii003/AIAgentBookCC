## 1. 概要

### 1.1 コンポーネントの目的

利用者のHuman-in-the-Loop承認（OK）取得後に申請書テンプレート（Excel/.xlsx）の所定セルへ申請情報を書き込み、申請書ドラフトファイルを `data/output/{session_id}/` に保存してパスを返す。交通費精算申請書（DATA-002）と経費精算申請書（DATA-003）でテンプレートファイルとセル入力仕様が異なるため、個別の関数として実装する。エージェント単独のLLM生成では、Excel形式（.xlsx）の正確なセル位置への書き込みを保証できないため、openpyxlライブラリを使用した確定的な生成を行う。

### 1.2 主要な責務

1. **テンプレートファイル読み込み**: `data/templates/交通費申請書_template.xlsx`（交通費）または `data/templates/経費精算申請書_template.xlsx`（経費）のExcelテンプレートを読み込む
2. **セルへの値書き込み**: 申請者名（B3）・申請日（B4）・各明細行（A{7+i}〜H{7+i}）・合計金額（H{7+n+2}）を書き込む
3. **申請書ドラフト保存**: 生成したExcelファイルを `data/output/{session_id}/` に保存する。出力ファイルパスはツール内部で `datetime.now().strftime("%Y%m%d%H%M%S")` を使用して自律的に生成する（LLMパラメータとして受け取らない）。`session_id` はツール関数内で `invocation_state` から取得する値である
4. **入力バリデーション**: `TravelApplicationFormInput` / `ExpenseApplicationFormInput` Pydanticモデルで収集済み申請情報の必須項目充足を検証する
5. **出力ディレクトリ管理**: `data/output/{session_id}/` ディレクトリはツール実行時に `os.makedirs(f"data/output/{session_id}", exist_ok=True)` で動的生成する
6. **申請者名・申請日の取得**: `@tool(context=True)` デコレーターを使用し、`invocation_state` から申請者名・申請日を取得する（LLMがツールパラメータとして渡す値ではない）

## 3. generate_travel_expense_formツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_travel_expense_form`

#### 説明（docstring）
```
交通費精算申請書（Excel/.xlsx）のドラフトを生成して data/output/{session_id}/ に保存する。

HumanApprovalHook（BeforeToolCallEvent）による承認OK取得後にのみ呼び出すこと。
出力ファイルパスはツール内部でタイムスタンプを使用して自律的に生成する。

申請者名・申請日・session_id はLLMがツールパラメータとして渡す値ではなく、
invocation_stateからツール関数内部で取得する（@tool(context=True) デコレーター使用）。

Args:
    items (list[dict]): 移動区間リスト（1件以上）。各要素は以下のキーを持つ辞書:
        - travel_date (str): 移動日（YYYY-MM-DD形式）
        - departure (str): 出発地（min_length=1）
        - destination (str): 目的地（min_length=1）
        - transport_type (str): 交通手段（"電車"/"バス"/"タクシー"/"飛行機"）
        - amount (int または str): 費用（円、0以上。"1,000円" 等の文字列も変換）
    business_purpose (str): 業務目的（min_length=1, BRL-20）

Returns:
    dict: 成功時は {"success": True, "file_path": str}、
          失敗時は {"success": False, "message": str}
```

---

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `items` | `list[dict]` | 移動区間リスト（1件以上。TravelItem の辞書リスト） | なし（必須） |
| `business_purpose` | `str` | 業務目的（BRL-20） | なし（必須） |

> **注意**: `applicant_name`・`application_date`・`session_id` はLLMがツールパラメータとして渡す値ではない。`@tool(context=True)` デコレーターを使用し、ツール関数内部で `invocation_state` から取得する。

**各要素のフィールド（items リスト内の各辞書）**:

| フィールド | 型 | 必須/任意 | 説明 |
|-----------|-----|----------|------|
| `travel_date` | `str` | 必須 | 移動日（YYYY-MM-DD形式。parse_dateで変換） |
| `departure` | `str` | 必須 | 出発地（min_length=1） |
| `destination` | `str` | 必須 | 目的地（min_length=1） |
| `transport_type` | `str` | 必須 | 交通手段（正規化後にLiteral["電車","バス","タクシー","飛行機"]） |
| `amount` | `int` または `str` | 必須 | 費用（円、ge=0。parse_amountで変換） |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ | 例 |
|-----------|--------|----------------|-----|
| `items` | 1件以上（model_validator(mode="after")で確認） | "申請明細が1件以上必要です。移動情報を入力してください。" | NG: `[]` |
| `items[*].amount` | ge=0の整数。"1,000円" 等はparse_amountで変換 | "金額のエラー: 0以上の整数で入力してください。" | OK: `200`, `"1,000円"` / NG: `-100` |
| `business_purpose` | min_length=1。strip()後に空文字チェック（BRL-20） | "業務目的が入力されていません。申請の業務目的を入力してください。" | OK: `"社内会議参加"` / NG: `""` |

##### 正規化処理（items[*].transport_type）

| 入力値 | 正規化後 | 処理内容 |
|--------|---------|---------|
| `"train"`, `"鉄道"`, `"地下鉄"` | `"電車"` | `TRANSPORT_TYPE_MAP` による変換（field_validator(mode="before")） |
| `"bus"` | `"バス"` | `TRANSPORT_TYPE_MAP` による変換 |
| `"taxi"`, `"cab"` | `"タクシー"` | `TRANSPORT_TYPE_MAP` による変換 |
| `"airplane"`, `"plane"` | `"飛行機"` | `TRANSPORT_TYPE_MAP` による変換 |

#### 3.2.2 出力

**戻り値の型**: `dict[str, bool | str]`

**説明**: 成功時は申請書ドラフトのファイルパスを含む辞書、失敗時はエラーメッセージを含む辞書を返す

**構造**:
```python
# 成功時
{
    "success": bool,     # True
    "file_path": str     # 生成された申請書ドラフトのファイルパス（例: "data/output/{session_id}/交通費精算申請書_20260428143000.xlsx"）
}

# 失敗時
{
    "success": bool,   # False
    "message": str     # ユーザー向けエラーメッセージ（日本語）
}
```

#### 3.2.3 例外

| 例外 | 発生条件 |
|------|---------|
| `ValidationError` | `TravelApplicationFormInput` のバリデーション失敗時。`generate()` メソッド内でキャッチし `{"success": False, "message": ...}` を返す（再送出しない） |
| `Exception` | 想定外の例外発生時。`generate()` メソッド内でキャッチし `{"success": False, "message": ...}` を返す（再送出しない） |
| `IOError` | ファイル保存メソッド（`_save_file()`）でのI/Oエラー。`(False, エラーメッセージ文字列)` を返す（再送出しない） |
| `PermissionError` | ファイル保存メソッド（`_save_file()`）での書き込み権限エラー。`(False, エラーメッセージ文字列)` を返す（再送出しない） |
| `Exception` | ファイル保存メソッド（`_save_file()`）での想定外エラー。`(False, エラーメッセージ文字列)` を返す（再送出しない） |

> **注意**: `generate()` メソッドでは `FileNotFoundError` を個別定義しない。テンプレートファイルの存在確認は `os.path.exists()` 事前チェックで行い、ファイルなしの場合は `{"success": False, "message": ...}` を返す。

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
invocation_state から申請者名・申請日・session_id を取得（@tool(context=True) 経由）
  ↓
入力バリデーション（TravelApplicationFormInput）
  - application_date: parse_dateバリデーターで datetime.date 型に変換
  - items: 各 TravelItem の全フィールドをバリデーション + model_validator で1件以上チェック
  - business_purpose: strip()後 min_length=1 チェック（BRL-20）
  ↓
ValidationError 発生？
  Yes → ErrorHandler.handle_validation_error(e) を呼び出しエラーメッセージを取得
        → {"success": False, "message": error_message} を返す
  No  → 次ステップへ
  ↓
テンプレートファイル存在チェック（os.path.exists() による事前チェック）
  "data/templates/交通費申請書_template.xlsx" が存在するか確認
  → ファイルなし → ErrorHandler.handle_data_inconsistency() → EX-06ログ記録（ERROR）
                 → {"success": False, "message": "申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。"} を返す
  ↓ （ファイルあり）
テンプレートファイル読み込み（openpyxl.load_workbook）
  ↓
セルへの値書き込み（移動区間明細: A列〜H列）
  ws["B3"] = applicant_name           # 申請者名（invocation_stateから取得）
  ws["B4"] = str(application_date)    # 申請日（YYYY-MM-DD形式の文字列）
  for i, item in enumerate(items):
      row = 7 + i
      ws[f"A{row}"] = i + 1                      # No（行番号）
      ws[f"B{row}"] = str(item.travel_date)       # 移動日
      ws[f"C{row}"] = item.departure              # 出発地
      ws[f"D{row}"] = item.destination            # 目的地
      ws[f"E{row}"] = item.transport_type         # 交通手段
      ws[f"F{row}"] = item.amount                 # 費用（int）
      ws[f"G{row}"] = business_purpose            # 業務目的
      ws[f"H{row}"] = ""                          # 承認状況（初期値：空欄）
  合計行: ws[f"H{7+n+2}"] = sum(item.amount for item in items)
  ↓
出力ディレクトリ生成（ツール実行時に動的生成）
  os.makedirs(f"data/output/{session_id}", exist_ok=True)
  ↓
出力ファイルパス生成（ツール内部）
  timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
  file_path = f"data/output/{session_id}/交通費精算申請書_{timestamp}.xlsx"
  ↓
ファイル保存（_save_file() メソッドで data/output/{session_id}/ ディレクトリへ）
  → IOError → ErrorHandler.handle_system_error(e) → EX-07ログ記録（ERROR、file_path付き）
             → (False, エラーメッセージ文字列) を返す
  → PermissionError → ErrorHandler.handle_system_error(e) → EX-07ログ記録（ERROR、file_path付き）
                    → (False, エラーメッセージ文字列) を返す
  → Exception → ErrorHandler.handle_unexpected_error(e) → EX-08ログ記録（CRITICAL、file_path付き）
              → (False, エラーメッセージ文字列) を返す
  ↓
_save_file() が (False, msg) を返した場合
  → {"success": False, "message": msg} を返す
  ↓
INFO ログ出力: "Travel expense form generated: {file_path}"
  ↓
{"success": True, "file_path": file_path} を返す
```

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
エラー発生
  ↓
エラー種別の判定
  ↓
ValidationError（EX-01/EX-06）
  → ErrorHandler.handle_validation_error(e) を呼び出し
  → ログ記録（ERRORレベル）
  → {"success": False, "message": error_message} を返す

テンプレートファイルなし（EX-06）
  → os.path.exists() 事前チェックで検出
  → ErrorHandler.handle_data_inconsistency() を呼び出し
  → ログ記録（ERRORレベル）
  → {"success": False, "message": "申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。"} を返す

ファイル書き込みエラー・IOError（EX-07、_save_file()内）
  → ErrorHandler.handle_system_error(e) を呼び出し
  → ログ記録（ERRORレベル、file_path付き）
  → (False, エラーメッセージ文字列) を返す

ファイル書き込みエラー・PermissionError（EX-07、_save_file()内）
  → ErrorHandler.handle_system_error(e) を呼び出し
  → ログ記録（ERRORレベル、file_path付き）
  → (False, エラーメッセージ文字列) を返す

ファイル書き込みエラー・Exception（EX-08、_save_file()内）
  → ErrorHandler.handle_unexpected_error(e) を呼び出し
  → ログ記録（CRITICALレベル、スタックトレース必須、file_path付き）
  → (False, エラーメッセージ文字列) を返す

想定外例外（EX-08、generate()内）
  → ErrorHandler.handle_unexpected_error(e) を呼び出し
  → ログ記録（CRITICALレベル、スタックトレース必須）
  → {"success": False, "message": "申し訳ありません。予期しないエラーが発生しました。システム管理者にご連絡ください。"} を返す
```

#### 3.4.2 エラーメッセージの詳細

| エラー種別 | 発生条件 | 対応 | ユーザー向けメッセージ |
|-----------|---------|------|---------------------|
| E-001（EX-06）: テンプレートファイルなし | `data/templates/交通費申請書_template.xlsx` が存在しない（os.path.exists()で検出） | ErrorHandler.handle_data_inconsistency()でEX-06ログ記録。セッション終了促し | "申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。" |
| E-002（EX-01/EX-06）: 必須項目不足 | 申請日・明細1件以上のいずれかが欠如（ValidationError） | ErrorHandler.handle_validation_error()でEX-01ログ記録 | バリデーション詳細に応じたメッセージ（例: "申請明細が1件以上必要です。移動情報を入力してください。"） |
| E-003（EX-07）: ファイル書き込みエラー・IOError | `data/output/{session_id}/` ディレクトリへの書き込み不可（IOError） | ErrorHandler.handle_system_error()でEX-07ログ記録（ERRORレベル、file_path付き）。_save_file()が(False, msg)を返す | "申し訳ありません。ファイルの生成に失敗しました。システム管理者にご連絡ください。" |
| E-004（EX-07）: ファイル書き込みエラー・PermissionError | `data/output/{session_id}/` ディレクトリへの書き込み権限なし（PermissionError） | ErrorHandler.handle_system_error()でEX-07ログ記録（ERRORレベル、file_path付き）。_save_file()が(False, msg)を返す | "申し訳ありません。ファイルの生成に失敗しました。システム管理者にご連絡ください。" |

| 例外 | 発生条件 |
|------|---------|
| `ValidationError` | `ExpenseApplicationFormInput` のバリデーション失敗時。`generate()` メソッド内でキャッチし `{"success": False, "message": ...}` を返す（再送出しない） |

### 4.3 ビジネスロジック

#### 4.3.1 処理フロー

```
開始
  ↓
invocation_state から申請者名・申請日・session_id を取得（@tool(context=True) 経由）
  ↓
入力バリデーション（ExpenseApplicationFormInput）
  - application_date: parse_dateバリデーターで datetime.date 型に変換
  - items: 各 ExpenseItem の全フィールドをバリデーション + model_validator で1件以上チェック
  - business_purpose: strip()後 min_length=1 チェック（BRL-20）
  ↓
ValidationError 発生？
  Yes → ErrorHandler.handle_validation_error(e) → EX-01ログ記録 → {"success": False, "message": error_message} を返す
  No  → 次ステップへ
  ↓
テンプレートファイル存在チェック（os.path.exists() による事前チェック）
  "data/templates/経費精算申請書_template.xlsx" が存在するか確認
  → ファイルなし → ErrorHandler.handle_data_inconsistency() → EX-06ログ記録
                 → {"success": False, "message": "申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。"} を返す
  ↓ （ファイルあり）
テンプレートファイル読み込み（openpyxl.load_workbook）
  ↓
セルへの値書き込み（経費明細: A列〜H列）
  ws["B3"] = applicant_name           # 申請者名（invocation_stateから取得）
  ws["B4"] = str(application_date)    # 申請日（YYYY-MM-DD形式の文字列）
  for i, item in enumerate(items):
      row = 7 + i
      ws[f"A{row}"] = i + 1                      # No（行番号）
      ws[f"B{row}"] = str(item.purchase_date)     # 購入日
      ws[f"C{row}"] = item.store_name             # 店舗名

  ↓
出力ディレクトリ生成（ツール実行時に動的生成）
  os.makedirs(f"data/output/{session_id}", exist_ok=True)
  ↓
出力ファイルパス生成（ツール内部）
  timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
  file_path = f"data/output/{session_id}/経費精算申請書_{timestamp}.xlsx"
  ↓
ファイル保存（_save_file() メソッドで data/output/{session_id}/ ディレクトリへ）
  → IOError → ErrorHandler.handle_system_error(e) → EX-07ログ記録（ERRORレベル、file_path付き）
             → (False, エラーメッセージ文字列) を返す
  → PermissionError → ErrorHandler.handle_system_error(e) → EX-07ログ記録（ERRORレベル、file_path付き）
                    → (False, エラーメッセージ文字列) を返す

  ↓

  → {"success": False, "message": msg} を返す
  ↓
INFO ログ出力: "Expense form generated: {file_path}"
  ↓

```

## 6. 補足情報

### 6.1 実装上の注意点

1. **例外の再送出禁止**
   - 両ツール関数内で ValidationError・IOError・PermissionError・その他例外を再送出しない
   - `generate()` メソッドの全例外は `{"success": False, "message": ...}` の dict 形式で返す
   - `_save_file()` メソッドの全例外は `(False, エラーメッセージ文字列)` のタプル形式で返す

2. **戻り値形式の統一**
   - ファイル保存メソッド（`_save_file()`）のエラー時戻り値は `(False, エラーメッセージ文字列)` のタプル形式とする
   - 申請書生成メソッド（`generate()`）のエラー時戻り値は `{"success": False, "message": エラーメッセージ文字列}` の dict 形式とする

3. **ファイル存在チェックの実装方針**
   - ファイル存在チェックは `os.path.exists()` による事前チェックで実装すること
   - `open()` 実行時の `FileNotFoundError` 捕捉との併用は禁止し、事前チェックを優先する

4. **例外クラスの定義方針**
   - ファイル保存メソッド（`_save_file()`）の例外クラスは `IOError`・`PermissionError`・`Exception` の3種類を個別に定義する（`IOError` を `PermissionError` に統合しない）
   - 申請書生成メソッド（`generate()`）の例外クラスは `ValidationError` と `Exception` の2種類のみとする（`FileNotFoundError` は定義しない）

5. **全エラーログへの file_path 付加**
   - ファイル保存に関わるエラーログにはすべて `file_path` を付加情報として含める

6. **申請者名・申請日・session_id の取得方法**
   - `@tool(context=True)` デコレーターを使用し、ツール関数内部で `invocation_state` から申請者名・申請日・session_id を取得する
   - `applicant_name`・`application_date`・`session_id` はLLMがツールパラメータとして渡す値ではないため、関数の引数一覧に含めない

7. **出力ファイルパスの自律生成**
   - ファイルパス（ファイル名）はツール内部で `datetime.now().strftime("%Y%m%d%H%M%S")` を使用して生成する
   - LLMから出力ファイルパスを引数として受け取る実装は禁止

8. **data/output/{session_id}/ ディレクトリの動的生成**
   - 出力ディレクトリはツール実行時に `os.makedirs(f"data/output/{session_id}", exist_ok=True)` で動的生成する
   - `session_id` はツール関数内で `invocation_state` から取得する値である
   - ベースディレクトリ設定値: `OUTPUT_DIR`: `"data/output"`（サブディレクトリは session_id で動的生成）

9. **Strands SDKのデコレーター**
   - `@tool(context=True)` デコレーターを使用し、invocation_stateから申請者名・申請日・session_id を取得する
   - モジュール: `tools/output_generator.py`

10. **HumanApprovalHookとの連携**
    - HumanApprovalHookは `BeforeToolCallEvent` で `generate_travel_expense_form` / `generate_expense_form` をブロック対象として登録する
    - 承認（OK）取得後にのみ本ツールが実行される（GRD-016、BRL-06）

---

### 6.2 パフォーマンス考慮事項

1. **テンプレートファイルの毎回読み込み**
   - テンプレートファイルはツール実行のたびに読み込む（静的キャッシュなし）
   - HumanApprovalHookによる承認後に1回のみ実行されるため、パフォーマンス上の問題はない

2. **べき等性**
   - 実行ごとに異なるタイムスタンプでファイルを生成するため冪等ではない
   - HumanApprovalHookによる承認後の1回のみ呼び出すことで重複生成リスクを管理する（実行制御方針 7.2節）

---

### 6.3 セキュリティ考慮事項

1. **出力パスのハードコード**
   - 出力先は `data/output/{session_id}/` ディレクトリに固定する。LLMから受け取ったパスを使用しない
   - パス操作によるディレクトリトラバーサルのリスクを排除する

2. **申請書自動提出禁止**
   - 本ツールはファイルを `data/output/{session_id}/` に保存するのみ（BRL-09）
   - 提出操作（APIコール・メール送信等）は実装しない

- 有効な交通費申請情報で `generate_travel_expense_form` を呼び出し、`success=True`・B3セルに申請者名・B4セルに申請日・A7〜H7+n行に明細（A列=No、B列=移動日、C列=出発地、D列=目的地、E列=交通手段、F列=費用、G列=業務目的、H列=空欄）・合計行が正しく書き込まれたExcelファイルが `data/output/{session_id}/` に生成されること
- ファイルパスがタイムスタンプを含む形式（`data/output/{session_id}/交通費精算申請書_YYYYMMDDHHMMSS.xlsx`）で生成されること
- 有効な経費申請情報で `generate_expense_form` を呼び出し、A列=No、B列=購入日、C列=店舗名、D列=品目、E列=経費区分、F列=金額、G列=業務目的、H列=空欄の形式でExcelファイルが `data/output/{session_id}/` に生成されること
- `expense_category="文房具"` を渡し `"事務用品費"` に正規化されて正常処理されること
- 申請者名・申請日が invocation_state から取得され、B3・B4セルに正しく書き込まれること

### 8.2 異常系テスト
- テンプレートファイルが存在しない状態でツールを呼び出し、`success=False`・エスカレーション案内メッセージが返ること（EX-06、os.path.exists()検出）
- 明細リストが空（`items=[]`）でツールを呼び出し、`success=False`・バリデーションエラーメッセージが返ること（EX-01）
- `business_purpose=""` の空文字でツールを呼び出し、`success=False`・業務目的エラーメッセージが返ること（EX-01）
- `data/output/{session_id}/` への書き込み権限がない状態でツールを呼び出し、`_save_file()` が `(False, msg)` を返し、最終的に `{"success": False, "message": ...}` が返ること（EX-07、PermissionError）
- `data/output/{session_id}/` でIOErrorが発生した場合、`_save_file()` が `(False, msg)` を返し、最終的に `{"success": False, "message": ...}` が返ること（EX-07、IOError）
- HumanApprovalHook承認前にツールが呼び出せないこと（HumanApprovalHookによるブロック）

### 8.3 性能テスト
- 10件の明細を含む申請情報でファイル生成が3秒以内に完了すること
- 重複実行（同じ申請情報で2回呼び出し）で異なるタイムスタンプのファイルが生成されること

### 8.4 境界値テスト
- `items` が1件の場合に正常にファイルが生成されること
- `amount=0` の明細が含まれる場合に正常にファイルが生成されること
- 出力ファイルパスがLLMパラメータ（引数）として受け取れないこと（セキュリティ確認）

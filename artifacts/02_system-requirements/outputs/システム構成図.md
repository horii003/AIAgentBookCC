## システム構成図（Mermaid）

```mermaid
flowchart TD
    User["👤 社員\n（一般社員）"]

    subgraph AgentSystem["AIエージェントシステム（申請AIエージェント）"]
        AG001["AG-001\n申請受付窓口エージェント\n（オーケストレーター）"]
        AG002["AG-002\n交通費精算申請エージェント\n（サブエージェント）"]
        AG003["AG-003\n経費精算申請エージェント\n（サブエージェント）"]
    end

    subgraph Tools["ツール群"]
        TOOL001["TOOL-001\n交通費計算ツール"]
        TOOL002["TOOL-002\n申請書生成ツール"]
    end

    subgraph DB["データストア（ファイルストレージ）"]
        DB_ROUTE["railway_routes.json\n（電車経路・運賃JSON: DATA-009）"]
        DB_FARE["fixed_fares.json\n（固定運賃JSON: DATA-010）"]
        FS["申請書ファイルストレージ\n（EXT-002）"]
        SS["セッションファイルストレージ\n（DATA-006）"]
    end

    LLM["LLMバックエンド\n（EXT-001：Claude等）"]
    KN["社内申請ルールナレッジ\n（システムプロンプト埋め込み）"]

    User -->|"申請者名・申請意図入力"| AG001
    AG001 -->|"委譲：交通費精算申請フロー"| AG002
    AG001 -->|"委譲：経費精算申請フロー"| AG003

    AG002 -->|"ツール呼び出し"| TOOL001
    AG002 -->|"ツール呼び出し"| TOOL002
    AG003 -->|"ツール呼び出し"| TOOL002

    TOOL001 -.->|"経路JSONファイル参照"| DB_ROUTE
    TOOL001 -.->|"固定運賃JSONファイル参照"| DB_FARE
    TOOL002 -.->|"申請書テンプレート取得・ファイル出力"| FS

    AG001 -.->|"LLM推論呼び出し"| LLM
    AG002 -.->|"LLM推論呼び出し"| LLM
    AG003 -.->|"LLM推論呼び出し"| LLM

    KN -.->|"システムプロンプト埋め込み"| AG001
    KN -.->|"システムプロンプト埋め込み"| AG002
    KN -.->|"システムプロンプト埋め込み"| AG003

    AG001 -.->|"セッション状態読み書き"| SS
    AG002 -.->|"セッション状態読み書き"| SS
    AG003 -.->|"セッション状態読み書き"| SS

    AG002 -->|"案内・チェック結果・申請書提示"| User
    AG003 -->|"案内・チェック結果・申請書提示"| User
    User -.->|"領収書画像提供（経費精算申請時）"| AG003
```

> **矢印凡例:**
> - 実線矢印（→）：処理の委譲・呼び出し・応答
> - 破線矢印（-.->）：データの参照・取得・埋め込み

# AIエージェント ガードレール詳細設計

> **参照元（システム要件定義資料）:**
> - artifacts/02_system-requirements/outputs/ガードレール要件定義.md（ガードレール実現要件 GRD-001〜GRD-015）

---
## 1. 概要

### 1.1 ガードレールの目的

AGエージェント（AG-001/AG-002/AG-003）が業務で利用される際の誤操作・誤判断・逸脱・リスクを防止するための制御を実現する。入力・出力・判断・実行の各段階においてコンテンツフィルタリング・PII検出・有害単語ブロックを適用し、業務上のガードレール要件（GRD-001〜GRD-015）を技術的に担保する。

### 1.2 主要な責務

1. **コンテンツフィルタリング**: 暴力・プロンプト攻撃・不正行為・ヘイト・性的コンテンツ・侮辱の入出力をブロックする
2. **PII検出・マスキング**: 個人情報（氏名・メールアドレス等）の出力を抑制する（GRD-011）
3. **有害単語ブロック**: AWS マネージド有害単語リストを適用し、業務申請と無関係な有害コンテンツをブロックする
4. **ブロック時のユーザーメッセージ返却**: ブロック時は技術的詳細を含まない日本語メッセージを社員に返す

### 1.3 適用範囲

- 対象エージェント: AG-001, AG-002, AG-003（全エージェント共通）
- 適用フェーズ: 入力受付時・会話・生成・実行
- 注: GRD-006〜GRD-015 のうち業務ロジック系（期限チェック・申請書提出禁止等）はエージェントのシステムプロンプト・ErrorHandler・HumanApprovalHook・LoopControlHook で実現。AWS Bedrock Guardrails では上記のコンテンツ安全性制御のみを担当する

### 1.4 基本情報

| 項目 | 内容 |
|------|------|
| ガードレール名 | `expense-agent-guardrail` |
| バージョン説明 | 交通費・経費精算申請エージェント用ガードレール v1.0 |
| リソースID | 要件上未定義（AWS コンソールまたは CLI で作成時に発行） |
| リージョン | ap-northeast-1（東京リージョン） |
| ガードレールプロファイル | STANDARD（標準フィルタリング層） |

## 2. ブロック時のメッセージ

- メッセージは日本語で表示する
- エラーコードや技術的な詳細はユーザーに見せない

### 入力ブロック時
```
その話題は有害または不適切である可能性があるため、情報を提供できません。
安全にお手伝いできる別の内容についてご質問ください。
```

### 出力ブロック時
```
そのような種類のコンテンツは、有害または不適切である可能性があるため生成できません。
他にお手伝いできることがあればお知らせください。
```

## 3. コンテンツフィルター

フィルタリング層: STANDARD

| カテゴリ | 入力強度 | 出力強度 | 入力アクション | 出力アクション |
|----------|----------|----------|----------------|----------------|
| VIOLENCE（暴力） | HIGH | HIGH | BLOCK | BLOCK |
| PROMPT_ATTACK（プロンプト攻撃） | HIGH | NONE | BLOCK | NONE |
| MISCONDUCT（不正行為） | HIGH | HIGH | BLOCK | BLOCK |
| HATE（ヘイト） | HIGH | HIGH | BLOCK | BLOCK |
| SEXUAL（性的コンテンツ） | HIGH | HIGH | BLOCK | BLOCK |
| INSULTS（侮辱） | MEDIUM | MEDIUM | BLOCK | BLOCK |

## 5. 機密情報ポリシー

### PII（個人識別情報）

| 種別 | 入力有効 | 出力有効 | アクション |
|------|----------|----------|------------|
| NAME（氏名） | false | true | ANONYMIZE |
| EMAIL（メールアドレス） | false | true | BLOCK |
| PHONE（電話番号） | false | true | BLOCK |
| ADDRESS（住所） | false | true | ANONYMIZE |

> **注**: 入力は申請者名・住所等の業務情報を正常入力として受け付けるため `false`（検出しない）に設定する。出力については意図しない個人情報の露出を防ぐため ANONYMIZE/BLOCK を適用する（GRD-011）。

### カスタム正規表現

なし

## 評価指標ID：MET-010
## 評価指標名：ツール選択精度

---

### 1. 基本情報

| 項目 | 内容 |
|---|---|
| 評価カテゴリ | 精度 |
| 評価観点 | 受付エージェントによる適切な専門エージェント選択の正確性 |
| 対象エージェント | AG-001 |
| 対象機能ID | FR-001, FR-002 |
| 対象会話フロー | CF-001, CF-002 |
| 対象成果物 | — |

---

### 2. 指標の目的・意味

- 受付エージェント（AG-001）が申請内容を正しく分類し、適切な専門エージェント（AG-002またはAG-003）を選択・振り分けできているかを測る
- 業務ゴール：誤った専門エージェントへの振り分けによる申請フロー誤動作の防止
- 精度が低下した場合はプロンプト改善またはナレッジ/マスタファイル更新のトリガーとする

---

|---|---|
| 定義式 | 正しい専門エージェントに振り分けられた件数 / 全評価件数 |
| 評価単位 | 会話（1申請フロー単位） |
| スコア範囲 | 0〜1（0%〜100%） |
| 高いほど良い | Yes |

---

### 4. 評価方法・測定手順

| 項目 | 内容 |
|---|---|
| 評価方法 | 自動スクリプト（振り分け先エージェントと正解ラベルの照合） |
| 評価データ | 申請内容テキスト + 正解振り分け先エージェントラベルのゴールドデータ |
| 実施手順 | テストシナリオを実行し、AG-001が選択した振り分け先エージェントを正解ラベルと照合して一致率を算出する |
| 評価者 | システム（自動） |

## 評価指標ID：MET-011
## 評価指標名：ゴール達成率

---

### 1. 基本情報

| 項目 | 内容 |
|---|---|
| 評価カテゴリ | 機能妥当性 |
| 評価観点 | 受付エージェントの振り分けによって会話フローのゴールが達成されたか |
| 対象エージェント | AG-001, AG-002, AG-003 |
| 対象機能ID | FR-001〜FR-011 |
| 対象会話フロー | CF-001〜CF-006 |
| 対象成果物 | OUT-002（申請書（下書き）） |

---

### 2. 指標の目的・意味

- 受付エージェントの振り分けを起点に、申請フロー全体のゴール（申請書（下書き）の生成完了、または適切なエスカレーション）が達成されているかを継続的に測る
- 業務ゴール：社員が申請手続きを完了できる確率の最大化（RQ-01、RQ-03）
- 達成率が低下した場合は振り分けロジック・情報収集フロー・申請書生成処理を総合的に見直す

---

### 3. 指標定義（定量）

#### 3.1 指標種別
- [x] 定量指標
- [ ] 定性指標

#### 3.2 指標定義

| 項目 | 内容 |
|---|---|
| 定義式 | ゴール達成件数 / 全評価件数 |
| 評価単位 | 会話（1申請フロー単位） |
| スコア範囲 | 0〜1（0%〜100%） |
| 高いほど良い | Yes |

> **ゴール定義：** 申請書（下書き）の生成完了（OUT-002）、または申請不可時の適切なエスカレーション案内（GRD-005、ABAC-004）が行われた状態

---

### 4. 評価方法・測定手順

| 項目 | 内容 |
|---|---|
| 評価方法 | 自動スクリプト（フロー完了ステータスと期待ゴールの照合） |
| 評価データ | 代表的な申請シナリオ（各会話フロー CF-001〜CF-006 を網羅） |
| 実施手順 | テストシナリオを実行し、フロー終了時の成果物・アクション結果と期待ゴールを照合して達成率を算出する |
| 評価者 | システム（自動） |

---

### 5. 入力データ・期待結果

| No | 入力条件 | 期待結果 | 備考 |
|---:|---|---|---|
| 1 | 交通費精算申請の完全なシナリオ（CF-003→CF-005） | 申請書（下書き）の生成完了 | OUT-002 |
| 2 | 経費精算申請の完全なシナリオ（CF-004→CF-005） | 申請書（下書き）の生成完了 | OUT-002 |
| 3 | 申請期限超過のシナリオ | 申請不可通知・エスカレーション案内 | ABAC-004、BRL-14 |
| 4 | 30ターン上限到達のシナリオ | エスカレーション案内（GRD-005） | GRD-005 |

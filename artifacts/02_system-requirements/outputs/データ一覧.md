## 2. データ詳細定義（データID単位）

### 2.1 データID：DATA-001（データ名：社内申請ルール）
- 種別：ナレッジ/ファイル
- 概要：BRL-01〜BRL-20, JD-01〜JD-16を含む申請種別・条件・申請期限・承認閾値・申請先・必要書類を定義した社内ルール文書
- 形式：テキスト/Markdown
- 保管場所：ファイルストレージ（ナレッジ）
- 更新頻度：不定期（社内規程改訂時）
- 参照方法：ファイル参照（エージェントがファイルを読み込みプロンプトに埋め込み参照）
- データ分類・機密区分：D-1（社内）/ 社内
- マスキング要件：不要
- 保持期間：要件上未定義
- 品質要件：最新版の規程が反映されていること。改訂後の即時反映を推奨
- 監査要件：更新者・更新日時を記録すること

#### スキーマ（要件定義レベル：代表項目のみ）

| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|
| 1 | 申請種別名 | string | ○ | 申請の種別名称 | 「交通費精算申請」 |
| 2 | 適用条件 | string | ○ | 申請種別が適用される条件 | 「業務上の交通費が発生した場合」 |
| 3 | 申請期限 | string | ○ | 経費発生日からの申請可能期間 | 「経費発生日から3ヶ月以内」 |
| 4 | 上長承認閾値 | number | ○ | 上長承認が必要となる金額 | 「交通費：10,000円超」「経費：5,000円超」 |
| 5 | 申請先 | string | ○ | 申請書の提出先 | 要件上未定義 |

---

### 2.2 データID：DATA-004（データ名：会話セッション情報）
- 種別：ファイル/セッション
- 概要：申請フロー全体の会話状態（STATE-001〜STATE-007）、収集済み申請情報、申請種別確定結果を保持するセッションデータ。セッション情報が存在する場合は復元可能（resume対応）
- 形式：JSON
- 保管場所：ファイルストレージ（セッション領域）
- 更新頻度：会話ターンごと
- 参照方法：ファイル参照（エージェントが直接参照・更新）
- データ分類・機密区分：D-2（機微）/ 機微
- マスキング要件：申請内容・申請情報等の個人情報をマスキング
- 保持期間：セッション終了後（保持期間は要件上未定義）
- 品質要件：セッション整合性が保たれること。複数ターンにわたる状態が正確に保持されること。セッション情報が存在する場合は復元可能なこと
- 監査要件：セッション開始・終了・状態変更を記録すること
- **制約：メモリや揮発性ストレージは使用しない**

#### スキーマ（要件定義レベル：代表項目のみ）

| No | 項目名 | 型（概略） | 必須 | 説明 | 例 |
|---:|---|---|---|---|---|
| 1 | セッションID | string | ○ | セッション識別子 | |
| 2 | 現在の状態ID | string | ○ | STATE-001〜STATE-007のいずれか | 「STATE-003」 |
| 3 | 申請種別 | string | △ | 確定した申請種別 | 「交通費精算申請」 |
| 4 | 収集済み申請情報 | object | △ | 収集済みの申請情報（D-007相当） | |
| 5 | 会話履歴参照 | reference | ○ | DATA-007（会話ログ）への参照 | |

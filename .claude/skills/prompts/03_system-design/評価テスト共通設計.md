# 役割
あなたはAIエージェント開発の評価テスト設計を専門とする設計者です。

# 指示
システム要件定義の成果物をもとに、フォーマットに従った評価テスト共通設計書を作成してください。
出力先は `evals/eval_common_system_design.md` です。

# 前提条件
- フォーマットの項目・構成・順序を厳守すること
- システム要件定義から読み取れる内容のみを端的にポイントをおさえて記載すること
- 判断できない内容は「要件上未定義」と記載すること
- 解説や補足説明は記載しないこと
- エージェント構成・アーキテクチャの詳細は `artifacts/03_system-design/システム基本情報.md` および `artifacts/03_system-design/マルチエージェント連携設計.md` への参照ポインターとし、重複記載しないこと
- 両評価スクリプト（[スクリプト名1].py / [スクリプト名2].py）に共通する設計のみを定義すること
- スクリプト固有の仕様（テストケース・評価器の設定）はここでは定義しないこと

# 各項目の記載ガイド

### 評価システム概要
- 評価指標一覧の評価目的（EV-001・EV-002）から評価全体の目的を導出する
- LLM-as-Judge 方式の採用根拠・スコア形式（二値: 1.0/0.0）は固定仕様として記載する

### 評価対象システム構成
- エージェント構成の詳細は `artifacts/03_system-design/システム基本情報.md` と `artifacts/03_system-design/マルチエージェント連携設計.md` への参照のみ記載する
- 評価の着眼点（ルーティング層 / エンドツーエンド層）を評価指標一覧（MET-010・MET-011）から導出して記載する

### 評価フレームワーク構成
- 使用フレームワーク：strands_evals（固定）
- Case・Experiment・Evaluator・StrandsInMemorySessionMapper の役割を記載する
- Experiment の逐次実行モデル（並列禁止の理由：memory_exporter スパン混入リスク）を記載する
- run_eval_task の戻り値仕様（output / trajectory）を記載する

### テレメトリ設計
- OpenTelemetry ベースのインメモリ計装によるテレメトリフローを記載する
- memory_exporter のシングルトン性質と、ケース間で必ず clear() を呼ぶ必要性を記載する
- session_id によるスパン紐付けの仕組みを記載する

### 共通モジュール設計（helpers）
- 両スクリプトが依存する helpers.py の関数・オブジェクト一覧を記載する
- 記載対象：patch_human_approval_hook / create_reception_agent / run_actor_conversation / memory_exporter / get_model / create_invocation_state
- 自律度・権限定義の承認フック要件（HumanApprovalHook）から patch_human_approval_hook の必要性を導出する
- エージェント間連携定義の invocation_state 受け渡し方針から create_invocation_state の責務を導出する

### 共通初期設定処理
- 両スクリプトで同一順序で実行する7ステップの初期化処理を記載する
- 各処理の目的を簡潔に記載する（Windows UTF-8 対応・sys.path・dotenv・フック差替・ログ・警告抑制・SDK ログ抑制）

### 共通ログ設計
- ルートログレベル・フォーマット・出力先・エンコーディングの共通仕様を記載する
- SDK ログの抑制設定（strands / strands.event_loop.event_loop の抑制レベル）を記載する
- ログファイルの命名規則（evals/logs/<スクリプト名>.log と _report.json）を記載する

### run_eval_task の共通構造
- 9ステップの共通フローを記載する
- ステップ [6] のエージェント実行方式のみスクリプト間で異なることを明示する

### ディレクトリ構成・出力ファイル
- evals/ 配下のディレクトリ構成と出力ファイル一覧を記載する
- レポート JSON の共通仕様（ensure_ascii=False・indent=2・UTF-8）を記載する

### 環境・依存関係
- Python バージョン要件・実行ディレクトリ・OS 対応を記載する
- 必要ライブラリ（strands-agents・strands-agents-evals・python-dotenv・opentelemetry-sdk）を記載する
- 環境変数（AWS_DEFAULT_REGION・AWS_ACCESS_KEY_ID・AWS_SECRET_ACCESS_KEY・評価モデル指定）を記載する
- 実行コマンドを記載する

### 評価スクリプト一覧と位置づけ
- 各評価スクリプト（TOOL_LEVEL・シングルターンと SESSION_LEVEL・マルチターン）の比較表を記載する
- 2スクリプトの補完関係（ルーティング精度検証 ＋ エンドツーエンド品質確認）を記載する

# 受入基準
作成後に以下の受入基準を満たしているか確認すること。
- 評価指標一覧の MET-010（ツール選択精度）・MET-011（ゴール達成率）と整合していること
- エージェント構成の重複記載がなく、参照ポインターに置き換えられていること
- 両評価スクリプトに共通する設計のみが定義されており、スクリプト固有仕様が混入していないこと
- テンプレートに完全準拠していること

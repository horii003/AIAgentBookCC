# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（会話フロー単位での委譲）
- 失敗時方針：エラーメッセージを返し人へエスカレーション案内
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（AG-001 → AG-002 または AG-003 のみ）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが並行/分担して成果を統合
- 本システムでの適用：なし（AG-001 → AG-002/AG-003 の委譲のみ。並行協調なし）

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを丸投げして結果を受け取る
- 本システムでの適用：LNK-001, LNK-002（AG-001がオーケストレーターとして申請種別に応じた専門エージェントへ委任）

### 3.3 イベント（Event-driven）
- 目的：状態変化を発火し、購読側が処理
- 本システムでの適用：なし（イベント駆動連携は要件上未定義）

## 4. 連携インタフェース定義（最小セット）

### 4.1 連携ID：LNK-001（AG-001 → AG-002 委譲）

#### 4.1.1 トリガー条件
- 発火条件：CF-001のStep 4またはCF-002のStep 2完了後、申請種別 = 交通費精算申請が確定した時点
- 事前条件（Pre-condition）：申請種別リスト（D-003）に「交通費精算申請」が確定されていること。社内申請ルール（DATA-001）が参照可能であること
- 事後条件（Post-condition）：AG-002が申請情報収集・申請書作成・チェックを完了し申請書ドラフト（D-008）と申請チェック結果（D-009）を返すこと

```json
{
  "request_id": "string（セッションID）",
  "task": "交通費精算申請フロー実行",
  "context": {
    "application_type": "交通費精算申請",
    "session_id": "string"
  },
  "constraints": {
    "auto_submit_prohibited": true
  },
  "approval_required": false
}
```

#### 4.1.3 出力スキーマ（概念レベル）
```json
{
  "request_id": "string（セッションID）",
  "status": "completed / error",
  "result": {
    "draft_id": "string",
    "check_result_id": "string"
  },
  "error_message": "string（エラー時のみ）"
}
```

## 5. エージェント依存関係・データフロー（概要）

```
利用者（R-EMP）
    │
    │ 申請内容（自由文）入力
    ▼
AG-001（申請受付窓口エージェント）
    │ 申請種別判定（BRL-01, BRL-02）
    │ 申請書・申請先提示（BRL-03）
    │
    ├─ [交通費精算申請確定] ─→ AG-002（交通費精算申請エージェント）
    │       LNK-001委譲         │ 移動情報収集（BRL-11〜BRL-15）
    │                           │ 申請書生成（BRL-05）→ D-008
    │                           │ ルールチェック（BRL-07〜BRL-09）→ D-009
    │                           │ 最終確認・提出促し（CF-007）
    │
    └─ [経費精算申請確定] ──→ AG-003（経費精算申請エージェント）
            LNK-002委譲         │ 経費情報収集（BRL-16〜BRL-17）
                                │ 申請書生成（BRL-05）→ D-008
                                │ ルールチェック（BRL-07〜BRL-09）→ D-009
                                │ 最終確認・提出促し（CF-007）
```

**注:** AG-001とAG-002/AG-003の間でセッション情報（DATA-004）を共有する。AG-002とAG-003は直接連携しない（独立した専門エージェント）。

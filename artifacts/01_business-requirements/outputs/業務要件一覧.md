## ターゲットユーザー

社内の一般社員（申請ルールや申請方法に詳しくない場合がある）

## 対象申請種別

本システムが対象とする申請は以下の2種類のみとする。

| 申請種別 | 申請書名 |
| -------- | -------- |
| 交通費精算申請 | 交通費精算申請書 |
| 経費精算申請 | 経費精算申請書 |

### 申請種別の判断不能ケース

申請内容から交通費精算申請・経費精算申請のいずれかを判断できない場合は、ユーザーに申請種別を確認する。

- ユーザーに「交通費精算申請」「経費精算申請」の選択肢を提示する
- ユーザーの選択に従って対応する申請フローへ誘導する

## 要求事項

- RQ-01: 申請内容を入力すると必要な申請種別・申請書・申請先を提示する
- RQ-02: 申請ルールを知らない社員でも正しく申請できる
- RQ-03: 不足情報を対話で確認しながら申請書を自動作成する
- RQ-04: 申請内容をチェックし、申請ミスや差し戻しを減らす
- RQ-05: 社内の申請ルールを参照して申請種別を自動判断する

### 申請種別判断の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| JDG-01 | 申請内容から申請種別を特定する | ユーザーが入力した申請内容（自由文）を解析し、「交通費精算申請」「経費精算申請」のいずれかを判断する | High |
| JDG-02 | 申請種別が判断できない場合はユーザーに選択肢を提示して確認する | 入力情報だけでは申請種別を確定できない場合、「交通費精算申請」「経費精算申請」の2択を提示し、ユーザーの選択に従って対応するフローへ誘導する | High |

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRV-01 | 移動情報を一区間ずつ収集する | 移動日・出発地・目的地・交通手段・費用を一区間ごとに収集する | High |
| TRV-02 | 対応交通手段は電車・バス・タクシー・飛行機の4種類とする | 上記4種類以外の交通手段は対象外とする | High |
| TRV-03 | 運賃データから交通費を自動計算する | 電車は経路テーブル検索、バス・タクシー・飛行機は固定運賃で算出する | High |
| TRV-04 | 申請は経費発生日から3ヶ月以内であること | 経費発生日から3ヶ月を超える申請は受け付けない | High |
| TRV-05 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | 申請時に上長承認済みであることの確認を行う | High |
| TRV-06 | 駅名を正規化する | 「渋谷駅」→「渋谷」のように、余分な接尾語を除去して正規化する | Medium |
| TRV-07 | すべての申請に業務目的の記載が必要 | 業務目的が未記載の場合は入力を促す | High |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXP-01 | 領収書画像から情報を自動抽出する | 店舗名・金額・日付・品目を自動抽出する | High |
| EXP-02 | 品目から経費区分を自動判断する | 事務用品費・宿泊費・資格精算費・その他経費の4区分で判断する | High |
| EXP-03 | 申請は経費発生日から3ヶ月以内であること | 経費発生日から3ヶ月を超える申請は受け付けない | High |
| EXP-04 | 経費が5,000円を超える場合は事前に上長の承認が必要 | 申請時に上長承認済みであることの確認を行う | High |
| EXP-05 | すべての申請に業務目的の記載が必要 | 業務目的が未記載の場合は入力を促す | High |

### 申請書自動作成の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| FRM-01 | 申請書の必須項目を自動入力する | 判断した申請種別の申請書に対し、収集した情報から必須項目を自動入力する | High |
| FRM-02 | 不足必須項目は対話で補完する | 申請書の必須項目が不足している場合、不足項目を特定して利用者に質問する | High |
| FRM-03 | 申請書を利用者に提示して確認を得る | 自動作成した申請書は利用者に内容確認させ、承認を得る | High |

### 申請チェックの業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| CHK-01 | 申請ルール適合性をチェックする | 申請内容が社内申請ルールに適合しているかを確認し、不適合箇所を指摘する | High |
| CHK-02 | 差し戻しリスクが高い場合は警告する | 承認者による差し戻しリスクがある項目（金額・期日・理由不備等）を事前に警告する | Medium |

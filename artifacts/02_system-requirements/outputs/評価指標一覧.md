# 評価指標定義書

## 評価指標ID：MET-001
## 評価指標名：ツール選択精度

---

## 1. 基本情報（👤）

| 項目 | 内容 |
|---|---|
| 評価カテゴリ | 精度（Accuracy） |
| 評価観点 | ツール選択精度（サブエージェント振り分け一致率） |
| 対象エージェント | AG-001 |
| 対象機能ID | FR-001, FR-002 |
| 対象会話フロー | CF-001 |
| 対象成果物 | OUT-001（申請種別・申請書・申請先の案内結果） |

---

## 2. 指標の目的・意味（👤）

- AG-001が社員の申請意図（自然文）から正しいサブエージェント（AG-002：交通費精算申請、AG-003：経費精算申請）に振り分けられているかを継続的に検証する
- 継続検証目的：受付エージェントが正しく専門エージェントに振り分けていることを定期的に確認する
- 業務ゴール：申請ルールを知らない社員でも正しい申請先に申請できる（RQ-01）
- 改善アクション：振り分け精度が低い場合はシステムプロンプトのナレッジを修正する

---

## 3. 指標定義（定量／定性）

### 3.1 指標種別
- [x] 定量指標
- [ ] 定性指標

---

### 3.2 指標定義（👤）

| 項目 | 内容 |
|---|---|
| 定義式 | 正解サブエージェント振り分け数 / 全テストケース数 × 100 |
| 評価単位 | テストケース（申請意図テキスト単位） |
| スコア範囲 | 0〜100（%） |
| 高いほど良い | Yes |

---

## 4. 評価方法・測定手順（👤）

| 項目 | 内容 |
|---|---|
| 評価方法 | 自動スクリプト（テストシナリオ実行） |
| 評価データ | 交通費精算申請・経費精算申請の代表的な申請意図テキストのテストシナリオ（ゴールドデータ） |
| 実施手順 | テストシナリオをAG-001に入力し、振り分け先エージェントと正解ラベルを比較する |
| 評価者 | システム（自動） |

---

## 5. 入力データ・期待結果（🤖）

| No | 入力条件 | 期待結果 | 備考 |
|---:|---|---|---|
| 1 | 「タクシー代を精算したい」 | AG-002（交通費精算申請）に振り分け | 明確な交通費意図 |
| 2 | 「ホテルの宿泊費を精算したい」 | AG-003（経費精算申請）に振り分け | 明確な経費意図 |
| 3 | 「精算したい」（曖昧な意図） | 選択肢提示（BRL-01） | 判断不可時の代替フロー |

## 評価指標ID：MET-002
## 評価指標名：ゴール達成率

---

## 1. 基本情報（👤）

| 項目 | 内容 |
|---|---|
| 評価カテゴリ | 機能妥当性（Functional Correctness） |
| 評価観点 | ゴール達成率（申請フロー全体完了率） |
| 対象エージェント | AG-001, AG-002, AG-003 |
| 対象機能ID | FR-001〜FR-013 |
| 対象会話フロー | CF-001〜CF-005 |
| 対象成果物 | OUT-008（チェック合格済み交通費精算申請書）、OUT-009（チェック合格済み経費精算申請書） |

---

## 2. 指標の目的・意味（👤）

- 申請フロー全体（CF-001〜CF-005）が正常に完了し、チェック合格済み申請書が社員に提示されるまでの達成率を測定する
- 業務ゴール：申請書作成フローのエンドツーエンド完了（RQ-02〜RQ-05）
- 改善アクション：ゴール達成率が低い場合は中断ポイントを分析し、会話フローまたはエージェントのプロンプトを修正する

---

## 3. 指標定義（定量／定性）

### 3.1 指標種別
- [x] 定量指標
- [ ] 定性指標

---

### 3.2 指標定義（👤）

| 項目 | 内容 |
|---|---|
| 定義式 | 申請フロー正常完了数（チェック合格済み申請書生成まで） / 全テストケース数 × 100 |
| 評価単位 | テストケース（申請フロー単位：交通費精算申請・経費精算申請それぞれ） |
| スコア範囲 | 0〜100（%） |
| 高いほど良い | Yes |

---

## 4. 評価方法・測定手順（👤）

| 項目 | 内容 |
|---|---|
| 評価方法 | 自動スクリプト（申請フロー全体のエンドツーエンドテストシナリオ実行） |
| 評価データ | 交通費精算申請・経費精算申請の代表的な申請フローシナリオ（ゴールドデータ） |
| 実施手順 | テストシナリオを実行し、チェック合格済み申請書が生成されるまで到達したかを検証する |
| 評価者 | システム（自動） |

---

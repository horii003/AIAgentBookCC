## 1. 目的・位置づけ

### 1.1 目的
- AIエージェントの動作・判断・実行を**事後に検証・説明可能**とする
- 実装不具合・設計不備・運用事故を**再現・追跡可能**とする
- 監査・内部統制・セキュリティ要求に対応する

### 1.2 適用範囲
- 対象エージェント：AG-001, AG-002, AG-003
- 対象業務／ユースケース：BIZ-01（申請種別案内）、BIZ-02（申請書自動作成）
- 対象成果物：OUT-001〜OUT-006
- 対象フェーズ：会話／判断／生成／実行／連携

---

## 2. ログの分類と基本方針（プロジェクト標準）

| 区分 | ログ種別 | 主目的 | 主な利用者 |
|---|---|---|---|
| 監査系 | 監査ログ | 説明責任・証跡 | 監査/管理 |
| 運用系 | 操作ログ | 利用状況把握 | 運用 |
| 技術系 | トレースログ | 不具合解析 | 開発/運用 |
| 技術系 | エラーログ | 障害検知 | 運用 |
| 技術系 | デバッグログ | 実装確認 | 開発 |

> 原則：
> - **監査ログとデバッグログは混在させない**
> - **内部推論（Chain-of-Thought）は保存しない**

---

### 6.2 エラー分類
- 業務ルール違反（ERR-001〜ERR-003）：申請期限超過・対応外交通手段・申請種別判断不能
- 技術エラー（ERR-004〜ERR-007）：ナレッジ/マスタファイル参照失敗・テンプレート取得失敗・申請書生成失敗・LLM画像読み取り失敗
- 予期しない例外（ERR-008）：未定義の例外全般

## 8. ログフォーマット標準

### 8.1 標準操作ログ（LOG-STD）フォーマット

```json
{
  "request_id": "string（必須）",
  "step_id": "string（必須）",
  "tool_call_id": "string（ツール実行時は必須）",
  "actor": "AG-001|AG-002|AG-003",
  "action_id": "string（処理識別子）",
  "result": "success|failure",
  "timestamp": "ISO8601形式"
}
```

### 8.2 高重要度操作ログ（LOG-HI）フォーマット（LOG-STDの拡張）

```json
{
  "request_id": "string（必須）",
  "step_id": "string（必須）",
  "tool_call_id": "string（ツール実行時は必須）",
  "actor": "AG-001|AG-002|AG-003",
  "action_id": "string（処理識別子）",
  "result": "success|failure",
  "timestamp": "ISO8601形式",
  "application_type": "交通費精算申請|経費精算申請",
  "amount": "number（申請金額。該当時のみ）",
  "collected_fields_summary": "string（収集項目名リスト。値は含まない）",
  "approver_id": "string（上長承認取得済みの場合）",
  "approval_timestamp": "ISO8601形式（上長承認取得済みの場合）"
}
```

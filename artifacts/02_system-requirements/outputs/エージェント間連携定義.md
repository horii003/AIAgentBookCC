# エージェント間連携一覧（協調・委譲・イベント）

## 1. 連携ポリシー（概要）
- 連携方式：同期（リクエスト/レスポンス）
- 連携単位：タスク（申請種別確定後に対応エージェントへ委譲）
- 失敗時方針：人へエスカレーション案内（CF-006）
- 相互呼び出し制約：循環呼び出し禁止、最大委譲深度：1（AG-001→AG-002/AG-003の1段のみ）、タイムアウト：要件上未定義

## 3. 連携種別の定義（記入ルール）

### 3.1 協調（Collaboration）
- 目的：複数エージェントが**並行/分担**して成果を統合
- 本システムでの適用：なし（AG-001→AG-002/AG-003の直列委譲構造のため）

### 3.2 委譲（Delegation）
- 目的：起点エージェントがサブタスクを**丸投げ**して結果を受け取る
- 本システムでの適用：LNK-001（AG-001→AG-002）、LNK-002（AG-001→AG-003）。申請種別確定後、AG-001が申請種別専用エージェントに全フロー（情報収集・申請書作成・チェック）を委譲する

### 3.3 イベント（Event-driven）
- 目的：状態変化（例：申請書生成完了）を発火し、購読側が処理
- 本システムでの適用：なし（要件上未定義）

## 4. 連携インタフェース定義（最小セット）

### 4.1 連携ID：LNK-001

#### 4.1.1 トリガー条件
- 発火条件：AG-001が申請種別を「交通費精算申請」に確定した（JD-01またはユーザー選択後）
- 事前条件（Pre-condition）：D-003（申請種別：交通費精算申請、申請書名、申請先）が確定していること
- 事後条件（Post-condition）：AG-002がD-008（チェック合格済み申請書）を生成し、社員が最終確認できる状態になっていること

#### 4.1.2 入力スキーマ（構造化推奨）
```json
{
  "request_id": "string",
  "task": "交通費精算申請フロー実行",
  "context": {
    "application_type": "交通費精算申請",
    "application_form": "交通費精算申請書",
    "application_destination": "要件上未定義",
    "applicant_name": "string（申請者名：AG-001が収集済み）",
    "intention_text": "string（申請意図テキスト）"
  },
  "constraints": {},
  "approval_required": false
```

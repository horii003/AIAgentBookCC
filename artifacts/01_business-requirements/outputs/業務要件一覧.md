## ターゲットユーザー

社内の一般社員（申請ルールや申請方法に詳しくない場合がある）

## 要求事項

- RQ-01: 社員が申請内容（目的・状況）を入力すると、交通費精算申請・経費精算申請のいずれかを提示する
- RQ-02: 申請ルールや申請方法が分からない社員でも正しく申請できる
- RQ-03: エージェントが不足情報を対話で確認しながら申請書を自動作成する
- RQ-04: 申請内容のチェックを行い、申請ミスや差し戻しを減らす

## 業務要件

| ID    | 業務要件 | 関連要求  | AI関与 | 優先度 |
| ----- | -------- | --------- | ------ | ------ |
| BR-01 | 社員の申請意図（「タクシー代を精算したい」「領収書を精算したい」等）から交通費精算申請・経費精算申請のいずれかを判断する | RQ-01 | 実行 | High |
| BR-02 | 申請種別に対応する申請書と申請先を社員に提示する | RQ-01 | 実行 | High |
| BR-03 | 申請に必要な情報が不足している場合、対話で不足情報を確認・収集する | RQ-02, RQ-03 | 実行 | High |

| BR-05 | 作成した申請書の内容をチェックし、誤りや不備を検出して申請者にフィードバックする | RQ-04 | 実行 | High |
| BR-06 | 社内申請ルールを参照して申請種別の判断根拠を提示する | RQ-01, RQ-02 | 提案 | Medium |

## 業務種別ごとの業務ルール

### 交通費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| TRANS-01 | 移動情報の収集単位 | 移動日・出発地・目的地・交通手段・費用を一区間ずつ収集する | High |
| TRANS-02 | 対応交通手段 | 電車・バス・タクシー・飛行機の4種類 | High |
| TRANS-03 | 運賃自動計算 | 電車は経路テーブル検索、バス/タクシー/飛行機は固定運賃で自動計算する | High |
| TRANS-04 | 申請期限 | 経費発生日から90日以内であること | High |
| TRANS-05 | 上長承認要否 | 交通費が10,000円を超える場合は事前に上長の承認が必要 | High |
| TRANS-06 | 駅名正規化 | 「渋谷駅」→「渋谷」等、入力された駅名から接尾語を除去して正規化する | Medium |
| TRANS-07 | 業務目的必須 | すべての申請に業務目的の記載が必要 | High |

### 経費精算申請の業務ルール

| ID | 業務ルール | 詳細 | 優先度 |
| -- | ---------- | ---- | ------ |
| EXP-01 | 領収書自動抽出 | 領収書画像から店舗名・金額・日付・品目を自動抽出する | High |
| EXP-02 | 経費区分自動判断 | 品目から経費区分（事務用品費/宿泊費/資格精算費/その他経費）を自動判断する | High |
| EXP-03 | 申請期限 | 経費発生日から90日以内であること | High |
| EXP-04 | 上長承認要否 | 経費が5,000円を超える場合は事前に上長の承認が必要 | High |
| EXP-05 | 業務目的必須 | すべての申請に業務目的の記載が必要 | High |

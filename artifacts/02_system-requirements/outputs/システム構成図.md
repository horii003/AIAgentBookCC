## 2. マルチエージェント連携図（Mermaid）

```mermaid
sequenceDiagram
    actor User as 利用者（R-EMP）
    participant AG001 as AG-001<br/>申請受付窓口エージェント

    participant AG003 as AG-003<br/>経費精算申請エージェント
    participant Tools as ツール層<br/>TOOL-001/002

    User->>AG001: 申請内容（自由文）入力

        AG001->>AG002: 委譲（LNK-001）

        AG002->>Tools: 交通費計算（TOOL-001）
        AG002->>Tools: 申請書生成（TOOL-002）
        AG002->>Data: ルールチェック（社内申請ルール参照/ファイル）
        AG002->>User: 申請書ドラフト・チェック結果提示
        AG002->>User: 最終確認・提出促し
    else 経費精算申請確定
        AG001->>AG003: 委譲（LNK-002）
        AG003->>User: 経費情報収集（対話・領収書）
        AG003->>Tools: 申請書生成（TOOL-002）
        AG003->>Data: ルールチェック（社内申請ルール参照/ファイル）
        AG003->>User: 申請書ドラフト・チェック結果提示
        AG003->>User: 最終確認・提出促し
    end

    User->>Data: 提出操作（人のみ）
```

## 3. 注記

- フロントエンド（チャットUI）は要件上未定義（システム化対象外）
- インフラ構成（クラウド/オンプレ）は要件上未定義
- LLMモデル・バックエンドサービスは要件上未定義
- 外部システム連携は要件上未定義（申請先部門・申請先システムが未定義）
- データ永続化はファイル永続化を使用する（RDBは使用しない）
- ナレッジベースはファイルを使用する（RAGは使用しない）
- セッション情報はファイルストレージ（JSON）で管理する（メモリ・揮発性ストレージは使用しない）
- 申請書テンプレート・ドラフト・提出済み申請書はExcel（.xlsx）形式で管理する
- 運賃データは電車経路（JSON）と固定運賃（JSON）の2ファイルで管理する

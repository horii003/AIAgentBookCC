## 評価指標ID：MET-001
## 評価指標名：申請種別判定正確性

### 1. 基本情報

| 項目 | 内容 |
|---|---|
| 評価カテゴリ | 精度（Accuracy） |
| 評価観点 | 申請種別判定正確性 |
| 対象エージェント | AG-001 |
| 対象機能ID | FR-001 |
| 対象会話フロー | CF-001, CF-002 |
| 対象成果物 | OUT-001 |

### 2. 指標の目的・意味

- 申請内容（自由文）から申請種別（交通費精算申請・経費精算申請）を正確に判定できているかを評価する
- 判定精度が低い場合、利用者が誤ったフローへ誘導され申請不備が発生する
- 判定誤りが多い場合はプロンプト・ルール参照の改善アクションに使用する

### 3. 指標定義

#### 3.1 指標種別
- [x] 定量指標

#### 3.2 指標定義

| 項目 | 内容 |
|---|---|
| 定義式 | 正しく判定した件数 ÷ 全評価ケース数（自動判定ケースのみ、CF-001対象） |
| 評価単位 | 会話セッション件 |
| スコア範囲 | 0.0〜1.0 |
| 高いほど良い | Yes |

### 4. 評価方法・測定手順

| 項目 | 内容 |
|---|---|
| 評価方法 | 自動スクリプト（テストシナリオ実行） |
| 評価データ | 交通費精算申請・経費精算申請・判断不能ケースを含むテストシナリオセット |
| 実施手順 | テストシナリオをAG-001に入力し、出力された申請種別とゴールドラベルを比較する |
| 評価者 | システム（自動） |

### 5. 入力データ・期待結果

| No | 入力条件 | 期待結果 | 備考 |
|---:|---|---|---|
| 1 | 電車・バスによる移動費用に関する申請内容 | 申請種別 = 交通費精算申請 | JD-01対応 |
| 2 | 書籍購入・宿泊費等の経費に関する申請内容 | 申請種別 = 経費精算申請 | JD-01対応 |
| 3 | 申請種別が特定できない曖昧な入力内容 | 2択選択肢の提示（BRL-02） | JD-02対応 |

### 6. 合格基準・閾値

| 項目 | 内容 |
|---|---|
| 合格条件 | 自動判定ケースの判定一致率が目標値以上 |
| 目標値 | 要件上未定義 |
| 最低許容値 | 要件上未定義 |
| 不合格時対応 | プロンプト改善・ルール参照精度向上 |

### 7. 評価頻度・タイミング

| フェーズ | 実施有無 |
|---|---|
| PoC | ○ |
| 開発中 | ○ |
| リリース前 | ○ |
| 運用中 | △ |

| 自律度 | 要求精度 |
|---:|---|

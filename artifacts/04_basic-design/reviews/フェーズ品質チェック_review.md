## 3. システム設計との整合性チェック

### 3.1 エージェント設計の整合性

| チェック項目 | 内容 | 判定 |
|------------|------|------|
| AG-001 HumanApprovalHook 非登録 | AG-001 基本設計書の共通設定で「承認フック: なし」と明記されていること | ✅ |
| AG-002/AG-003 HumanApprovalHook 登録 | AG-002/AG-003 基本設計書の共通設定で「承認フック: あり (target: TOOL-002)」と明記されていること | ✅ |
| 全エージェント LoopControlHook 登録 | AG-001/AG-002/AG-003 全てで「ループ制御: 最大10回 (LoopControlHook)」が設定されていること | ✅ |
| AG-001 静的プロンプト | AG-001 の「システムプロンプトの生成方式」が「静的定数」であること（日付不要） | ✅ |
| AG-002/AG-003 動的プロンプト | AG-002/AG-003 の「システムプロンプトの生成方式」が「動的生成関数」であること（BRL-11の90日期限チェック用） | ✅ |
| SlidingWindow 設定値 | AG-001=30、AG-002=20、AG-003=15（共通設定方針.md 2.2準拠） | ✅ |
| AG-001 TOOL-002 実行不可 | AG-001 の非責務に「申請書の生成（TOOL-002）」が記載されていること | ✅ |
| AG-001 特殊コマンド | AG-001 の§10.2に `reset`/`リセット`/`最初から` コマンドが定義されていること | ✅ |
| AG-001 対話回数上限 | AG-001 の§4.2・§8.1に対話回数上限30回（SlidingWindow(30)と統一）が設定されていること | ✅ |
| AG-002/AG-003 ツール割り当て方針 | AG-002/AG-003 の§6.1に業務機能特化ツールのみ使用する方針が明記されていること | ✅ |
| AG-002/AG-003 エラー要約返却 | AG-002/AG-003 の§7.2（プロンプト）と§9.1（エラー対応）にシステム系エラー時の呼び出し元へ要約返却方針が明記されていること | ✅ |
| APR-001承認とTOOL-002実行の分離 | AG-002/AG-003 の§4.1・§4.2に「ドラフト提示後にAPR-001承認が1回のみ発生し、承認後にはじめてTOOL-002を呼び出す」設計が明記されていること | ✅ |

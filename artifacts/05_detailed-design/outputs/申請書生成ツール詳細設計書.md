## 1. 概要

### 1.1 コンポーネントの目的

利用者のHuman-in-the-Loop承認（OK）取得後に、申請書テンプレート（Excel/.xlsx）の所定セルへ申請情報を書き込み、申請書ドラフトファイルを `data/output/{session_id}/` に保存してパスを返す。交通費精算申請書と経費精算申請書でテンプレートファイルとセル入力仕様が異なるため、`generate_transport_expense_form` と `generate_expense_form` の2関数として実装する。LLM単独ではExcel（.xlsx）形式の特定セル位置への正確な書き込みを保証できないため、openpyxlライブラリを使用した確定的な生成を行う。

### 1.2 主要な責務

- **テンプレートファイル読み込み**: `data/templates/交通費申請書_template.xlsx`（交通費）または `data/templates/経費精算申請書_template.xlsx`（経費）を読み込む
- **申請者名・申請日の取得**: LLMがパラメータとして渡すのではなく、ツール関数内部で `invocation_state` から取得する
- **セルへの値書き込み**: 申請者名（B3）・申請日（B4）・各明細行・合計金額を書き込む
- **申請書ドラフト保存**: 生成したExcelファイルを `data/output/{session_id}/` ディレクトリに保存する
- **入力バリデーション**: 専用Pydanticモデルで収集済み申請情報の必須項目充足を検証する

## 3. generate_transport_expense_formツール設計

### 3.1 基本情報

#### ツール名（関数名）
`generate_transport_expense_form`

#### 説明（docstring）
```
交通費精算申請書ドラフト（Excel/.xlsx）を生成する。

Human-in-the-Loop承認（BeforeToolCallEvent OK）取得後にのみ呼び出すこと。
収集済み交通費申請情報を申請書テンプレートの所定セルに書き込み、data/output/{session_id}/ に保存する。
申請者名・申請日はツール関数内部で invocation_state から取得する（LLMがパラメータとして渡さない）。

Args:
    segments (list[dict]): 移動区間リスト（1件以上）。各要素は以下のキーを持つ辞書:
        - no (int): 行番号
        - transport_date (str): 移動日（YYYY-MM-DD形式）
        - departure (str): 出発地
        - destination (str): 目的地
        - transport_type (str): 交通手段
        - amount (int): 費用（円）
        - business_purpose (str): 業務目的
    business_purpose (str): 申請全体の業務目的。

Returns:
    dict: {
        "success": bool,
        "file_path": str,  # 成功時: 生成したファイルのパス
        "message": str     # 失敗時: ユーザー向けエラーメッセージ
    }
```

### 3.2 インターフェース設計

#### 3.2.1 入力

**パラメータ定義（LLMが渡すパラメータ）**:

| 引数名 | 型 | 説明 | デフォルト値 |
|--------|-----|------|--------------|
| `segments` | `list[dict]` | 移動区間リスト（1件以上） | なし（必須） |
| `business_purpose` | `str` | 業務目的（BRL-20） | なし（必須） |

**invocation_stateから取得する値（LLMがパラメータとして渡さない）**:

| キー | 説明 |
|------|------|
| `applicant_name` | 申請者名。ツール関数内部で `tool_context.invocation_state.get("applicant_name", "")` で取得 |
| `application_date` | 申請日。ツール関数内部で `tool_context.invocation_state.get("application_date", "")` で取得 |
| `session_id` | セッションID。出力ディレクトリ `data/output/{session_id}/` の生成に使用 |

**交通費精算申請書 移動区間明細（segments 各要素のフィールド）**:

| 列 | フィールド名 | 型 | 必須/任意 | セル |
|----|------------|-----|-----------|------|
| A | `no` | `int` | 必須 | No（行番号） |
| B | `transport_date` | `str` | 必須 | 移動日（YYYY-MM-DD形式） |
| C | `departure` | `str` | 必須 | 出発地 |
| D | `destination` | `str` | 必須 | 目的地 |
| E | `transport_type` | `str` | 必須 | 交通手段 |
| F | `amount` | `int` | 必須 | 費用（円, ge=0） |
| G | `business_purpose` | `str` | 必須 | 業務目的 |
| H | （承認状況） | - | 初期値: 空欄 | 承認状況（初期値：空欄） |

##### バリデーションルール詳細

| フィールド | ルール | エラーメッセージ |
|-----------|--------|-----------------|
| `applicant_name`（invocation_state） | `min_length=1`、空文字禁止 | `"申請者名が設定されていません。"` |
| `application_date`（invocation_state） | YYYY-MM-DD形式 | `"申請日の形式が不正です。YYYY-MM-DD形式で入力してください。"` |
| `segments` | 1件以上の要素を含むこと | `"移動区間が入力されていません。少なくとも1件の移動区間を入力してください。"` |
| `segments[*].transport_date` | YYYY-MM-DD形式 | `"移動日の形式が不正です。"` |
| `segments[*].departure` | `min_length=1` | `"出発地が入力されていません。"` |
| `segments[*].destination` | `min_length=1` | `"目的地が入力されていません。"` |
| `segments[*].transport_type` | `min_length=1` | `"交通手段が入力されていません。"` |
| `segments[*].amount` | `int`, `ge=0` | `"費用は0以上の整数で入力してください。"` |
| `business_purpose` | `min_length=1`。前後空白はstrip() | `"業務目的が入力されていません。申請の業務目的を入力してください。"` |

#### 3.2.2 出力

**戻り値の型**: `dict`

**構造**:
```python
# 成功時
{
    "success": bool,   # True
    "file_path": str   # 例: "data/output/20260323153045_a1b2c3d4/transport_expense_20260428123456.xlsx"
}

# 失敗時（generate()メソッドのエラー時）
{
    "success": bool,   # False
    "message": str     # ユーザー向けエラーメッセージ
}
```

#### 3.2.3 例外

| 例外 | 発生条件 |
|------|---------|
| なし（ツール関数は例外を raise しない） | ValidationError・PermissionError・Exception をすべて内部でキャッチし、`{"success": False, "message": ...}` を返す |

> **実装上の注意**: 例外を raise せず、必ず戻り値としてエラー情報を返す。

### 3.3 ビジネスロジック

#### 3.3.1 処理フロー

```
開始
  ↓
invocation_state から applicant_name・application_date・session_id を取得
  ↓
segments の必須キーガード確認
  ↓
  不足キーあり → {"success": False, "message": "移動区間データに必須項目が不足しています。不足項目: {不足キー名}"} を返す
  ↓
入力バリデーション（TransportApplicationFormInput.model_validate()）
  ↓
  ValidationError → 呼び出し元でログ出力 → ErrorHandler.handle_validation_error(e) → {"success": False, "message": ...} を返す
  ↓
テンプレートファイル存在チェック（os.path.exists()）
  ↓
  ファイル未存在 → 呼び出し元でWARNINGログ出力 → {"success": False, "message": "申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。"} を返す（_save_file メソッドの (False, str) 形式で内部処理）
  ↓
テンプレートファイル読み込み（openpyxl.load_workbook）
  ↓
セルへの値書き込み
  - ws["B3"] = applicant_name
  - ws["B4"] = application_date
  - for i, item in enumerate(validated_input.items):
      ws[f"A{7+i}"] = i + 1  # No（行番号）
      ws[f"B{7+i}"] = item.transport_date.isoformat()
      ws[f"C{7+i}"] = item.departure
      ws[f"D{7+i}"] = item.destination
      ws[f"E{7+i}"] = item.transport_type
      ws[f"F{7+i}"] = item.amount
      ws[f"G{7+i}"] = item.business_purpose
      # H列（承認状況）は空欄のまま
  - ws[f"F{7+n+2}"] = sum(item.amount for item in validated_input.items)  # 合計金額
  ↓
出力ファイルパスの生成
  - output_dir = f"data/output/{session_id}/"
  - timestamp = datetime.now().strftime("%Y%m%d%H%M%S")
  - file_path = f"{output_dir}transport_expense_{timestamp}.xlsx"
  ↓
出力ディレクトリの存在確認・生成（os.makedirs, exist_ok=True）
  ↓
ファイル保存（_save_file メソッド）
  - 成功: (True, file_path)
  - IOError: (False, エラーメッセージ)
  - PermissionError: (False, エラーメッセージ)
  - Exception: (False, エラーメッセージ)
  ↓
ファイル保存結果確認
  - (True, file_path): {"success": True, "file_path": file_path} を返す
  - (False, message): {"success": False, "message": message} を返す
```

#### 3.3.3 データの検証方法

**Pydanticモデル**: `TransportApplicationFormInput`

**検証内容**:
- `applicant_name`: `min_length=1`, `max_length=100`。前後空白はstrip()で除去（invocation_stateから取得後に検証）
- `application_date`: `parse_date` field_validator（mode="before"）で `datetime.date` 型に変換
- `items`: `list[TransportItem]`。`model_validator(mode="after")` で1件以上であることを確認
- `business_purpose`: `min_length=1`。前後空白はstrip()で除去

### 3.4 エラーハンドリング

#### 3.4.1 エラー処理フロー

```
エラー発生
  ↓
ValidationError（TransportApplicationFormInput）
  ↓
  呼び出し元でERRORログ出力
  → ErrorHandler.handle_validation_error(e)
  → {"success": False, "message": ユーザー向けメッセージ} を返す（generate()の戻り値形式）
  ↓
テンプレートファイル未存在
  ↓
  os.path.exists()で事前チェック → WARNINGログ出力（file_pathを付加情報として含める）
  → _save_file が (False, エラーメッセージ) を返す
  → {"success": False, "message": ...} を返す（generate()の戻り値形式）
  ↓
ファイル書き込みエラー（IOError）
  ↓
  呼び出し元でERRORログ出力（file_pathを付加情報として含める）
  → ErrorHandler.handle_file_save_error(e)
  → _save_file が (False, エラーメッセージ) を返す（_save_fileメソッドの戻り値形式）
  → {"success": False, "message": ...} を返す（generate()の戻り値形式）
  ↓
ファイル書き込みエラー（PermissionError）
  ↓
  呼び出し元でERRORログ出力（file_pathを付加情報として含める）
  → ErrorHandler.handle_file_save_error(e)
  → _save_file が (False, エラーメッセージ) を返す
  ↓
想定外の例外（Exception）
  ↓
  呼び出し元でERRORレベル（exc_info=True）ログ出力（file_pathを付加情報として含める）
  → ErrorHandler.handle_unexpected_error(e)
  → {"success": False, "message": "申し訳ありません。予期しないエラーが発生しました。システム管理者にご連絡ください。"} を返す（generate()の戻り値形式）
```

#### 3.4.2 例外テーブル

| 例外クラス | 発生条件 | 戻り値形式 | ユーザー向けメッセージ |
|-----------|---------|-----------|---------------------|
| `ValidationError` | TransportApplicationFormInput バリデーション失敗 | `{"success": False, "message": str}` | `"申請情報に不足している項目があります。{フィールド名}を入力してください。"` |
| `IOError` | ファイル書き込みアクセスエラー | `(False, str)` | `"申し訳ありません。申請書ファイルの保存に失敗しました。システム管理者にご連絡ください。"` |
| `PermissionError` | ファイル書き込み権限エラー | `(False, str)` | `"申し訳ありません。申請書ファイルの保存に失敗しました。ファイルの書き込み権限を確認してください。"` |
| `Exception`（generate()内） | generate()内の想定外エラー | `{"success": False, "message": str}` | `"申し訳ありません。予期しないエラーが発生しました。システム管理者にご連絡ください。"` |

> **注意**: ファイル保存メソッド（`_save_file`）のエラー時戻り値は `(False, エラーメッセージ文字列)` のタプル形式。申請書生成メソッド（`generate()`）のエラー時戻り値は `{"success": False, "message": エラーメッセージ文字列}` の dict 形式。`FileNotFoundError` は `generate()` 例外クラスとして定義しない（テンプレートファイルは `os.path.exists()` で事前チェック）。

#### 3.4.3 エラーメッセージ詳細テーブル

| エラー種別 | 発生条件 | ログレベル | ログ付加情報 | 戻り値形式 | ユーザー向けメッセージ |
|-----------|---------|-----------|------------|-----------|---------------------|
| E-001（テンプレート未存在） | テンプレートファイルが存在しない（os.path.exists() で検出） | WARNING | `file_path` | `(False, str)` | `"申し訳ありません。申請書テンプレートが見つかりません。システム管理者にご連絡ください。"` |
| E-002（バリデーションエラー） | ValidationError（必須項目欠如・明細0件） | ERROR | `file_path` | `{"success": False, "message": str}` | `"申請情報に不足している項目があります。{不足項目名}を入力してください。"` |
| E-003（IOError） | ファイルアクセスエラー | ERROR | `file_path` | `(False, str)` | `"申し訳ありません。申請書ファイルの保存に失敗しました。システム管理者にご連絡ください。"` |
| E-004（PermissionError） | ファイル権限エラー | ERROR | `file_path` | `(False, str)` | `"申し訳ありません。申請書ファイルの保存に失敗しました。ファイルの書き込み権限を確認してください。"` |

```python
{
    "success": bool,   # True
    "file_path": str   # 例: "data/output/20260323153045_a1b2c3d4/expense_20260428123456.xlsx"
}

# 失敗時（generate()のエラー時）
{
    "success": bool,   # False
    "message": str     # ユーザー向けエラーメッセージ
}
```

**セル書き込み**（generate_transport_expense_form との差分のみ）:
  - `ws["B3"] = applicant_name`
  - `ws["B4"] = application_date`
  - for i, item in enumerate(items):
    - A{7+i}=no, B{7+i}=purchase_date, C{7+i}=store_name, D{7+i}=item_name, E{7+i}=expense_category, F{7+i}=amount, G{7+i}=business_purpose
    - H{7+i} は空欄のまま（承認状況の初期値）
  - `ws[f"F{7+n+2}"] = sum(item.amount for item in items)` （合計金額）

**例外テーブル**（generate_transport_expense_form と同じクラス定義）:
- `ValidationError`・`IOError`・`PermissionError`・`Exception` の4種類のみとする（`FileNotFoundError` は定義しない）
- `_save_file` メソッドの例外: `IOError`・`PermissionError`・`Exception` の3種類を個別に定義する

**テンプレートパス**: `"data/templates/経費精算申請書_template.xlsx"`

**出力ファイルパス**: `f"data/output/{session_id}/expense_{timestamp}.xlsx"`

**エラーハンドリング・ログ出力**（ログプレフィックスを `[TL-002b]` に変更した上で generate_transport_expense_form と同様。全エラーログに `file_path` を付加情報として含める）

## 6. 補足情報

### 6.1 実装上の注意点

1. **申請者名・申請日・セッションIDは invocation_state から取得**
   - `applicant_name`・`application_date`・`session_id` はLLMがパラメータとして渡す値ではなく、ツール関数内部で `invocation_state` から取得する
   - `@tool(context=True)` デコレータを使用する

2. **ファイル存在チェックは os.path.exists() による事前チェック**
   - テンプレートファイルの存在確認は `os.path.exists()` で事前チェックする
   - `open()` 実行時の `FileNotFoundError` 捕捉との併用は禁止し、事前チェックを優先する

3. **例外の raise 禁止**
   - ツール関数内で例外を raise しない
   - `_save_file` メソッドは `IOError`・`PermissionError`・`Exception` を個別にキャッチして `(False, エラーメッセージ)` を返す
   - `generate()` メソッドは `ValidationError`・`Exception` のみをキャッチして `{"success": False, "message": ...}` を返す（`FileNotFoundError` は定義しない）

4. **全エラーログに file_path を付加情報として含める**
   - ログ出力時は `file_path` を付加情報として含めることで問題のファイル特定を容易にする

5. **出力ファイルパスはツール内部で生成**
   - `data/output/{session_id}/` ディレクトリ配下に `datetime.now().strftime("%Y%m%d%H%M%S")` を使用してファイル名を生成する

6. **HumanApprovalHookとの連携**
   - HumanApprovalHookはBeforeToolCallEventで `generate_transport_expense_form` と `generate_expense_form` の両関数をブロック対象として登録する

7. **output/ ディレクトリの自動生成**
   - `data/output/{session_id}/` ディレクトリが存在しない場合は `os.makedirs(..., exist_ok=True)` で自動生成する

---

### 6.2 パフォーマンス考慮事項

1. **Excelテンプレートの都度読み込み**
   - テンプレートファイルは各ツール呼び出し時に読み込む。呼び出し頻度が低いためキャッシュは不要

2. **大量明細への対応**
   - 明細件数に上限は設けないが、テンプレートの行数を超える場合はセル書き込みが正常に機能しない可能性がある。業務要件上の明細件数は通常数件〜十数件を想定する

---

### 6.3 セキュリティ考慮事項

1. **出力先パスのハードコード**
   - 出力先ディレクトリを `data/output/{session_id}/` に固定する。LLMからパスを受け取ることを禁止する

2. **テンプレートパスのハードコード**
   - テンプレートファイルパスを `data/templates/` 配下に固定する

## 8. テスト観点

### 8.1 機能テスト
- 有効な交通費申請情報で `generate_transport_expense_form` を呼び出し、`data/output/{session_id}/transport_expense_*.xlsx` が生成されること
- B3に申請者名・B4に申請日・A7以降に明細（No/移動日/出発地/目的地/交通手段/費用/業務目的）・H列が空欄・合計金額が書き込まれること
- 有効な経費申請情報で `generate_expense_form` を呼び出し、`data/output/{session_id}/expense_*.xlsx` が生成されること
- `applicant_name`・`application_date` が invocation_state から取得されること（LLMのパラメータとして渡されないこと）
- `data/output/{session_id}/` ディレクトリが存在しない状態で呼び出した際に、ディレクトリが自動生成されてファイルが保存されること

### 8.2 異常系テスト
- テンプレートファイルが存在しない場合に `{"success": False, "message": ...}` が返ること（E-001、WARNINGログ出力）
- 必須項目（業務目的等）が空文字の場合に `{"success": False, "message": ...}` が返ること（E-002）
- segments/items が空リストの場合に `{"success": False, "message": ...}` が返ること（model_validator）
- segments 内の辞書に必須キーが不足している場合に適切なエラーメッセージが返ること
- `data/output/` への書き込み権限がない場合に `_save_file` が `(False, エラーメッセージ)` を返すこと（E-003/E-004）
- IOError が `_save_file` で `(False, str)` として返されること（`{"success": False}` ではなく tuple 形式であること）
- PermissionError が `_save_file` で `(False, str)` として返されること
- generate() 内の Exception が `{"success": False, "message": ...}` として返されること

### 8.3 性能テスト
- 明細10件の申請書生成が5秒以内に完了すること

### 8.4 境界値テスト
- 明細1件（最小件数）で正常に動作すること
- 経費区分の別表記（`"文房具"` → `"事務用品費"`, `"ホテル"` → `"宿泊費"` 等）が正規化されること
- 金額が文字列形式（`"1,000円"`, `"500"` 等）で渡された場合に `parse_amount` で整数に変換されること

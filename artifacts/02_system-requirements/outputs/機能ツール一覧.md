### 3.1 ツールID：TOOL-001（ツール名：交通費計算ツール）

#### 3.1.1 概要

- 目的：移動区間情報（交通手段・出発地・目的地）と運賃データを照合して交通費を自動計算する
- 対象データ：経路テーブルデータ（電車）、固定運賃データ（バス・タクシー・飛行機）
- 副作用（更新有無）：なし（参照のみ）

#### 3.1.2 入力（概念）

| 入力項目 | 説明 | 必須/任意 |
|---------|------|-----------|
| 交通手段 | 電車・バス・タクシー・飛行機のいずれか | 必須 |
| 出発地 | 正規化済みの出発地名（電車の場合は駅名） | 必須 |
| 目的地 | 正規化済みの目的地名（電車の場合は駅名） | 必須 |

> 入力パラメータの詳細（型・バリデーションルール・キー名）はツール基本設計・詳細設計フェーズで定義する

#### 3.1.3 出力（概念）

| 出力項目 | 説明 |
|---------|------|
| 計算済み運賃 | 運賃データから算出した交通費の金額 |
| 計算可否フラグ | 自動計算可否（経路テーブル未存在の場合は手動入力が必要な旨を示す） |

> 出力スキーマの詳細（キー名・型・構造）はツール詳細設計フェーズで定義する

#### 3.1.4 失敗時動作

| 失敗パターン | 動作 |
|---|---|
| 経路テーブルに該当経路が存在しない（電車） | 自動計算不可を返却。呼び出し元エージェントが社員に手動入力を促す |
| 固定運賃データが存在しない（バス・タクシー・飛行機） | 自動計算不可を返却。呼び出し元エージェントが社員に手動入力を促す |
| データファイルアクセスエラー | エラーを返却。呼び出し元エージェントが社員にエラー通知と管理部門エスカレーション案内を提示する |

#### 3.1.5 承認・権限

| 項目 | 内容 |
|---|---|
| 承認要否 | 不要（参照のみ） |
| 権限ID | なし |
| 自律度 | Lv3（実行） |
| 呼び出し可能エージェント | AG-002 |

### 3.2 ツールID：TOOL-002（ツール名：申請書生成ツール）

#### 3.2.1 概要

- 目的：収集済み申請情報（経費精算または交通費精算）と申請書テンプレートからExcel形式の申請書（下書き）を生成する。申請種別に応じてDATA-002（経費精算テンプレート）またはDATA-003（交通費精算テンプレート）を使用する
- 対象データ：申請書作成用情報（D-007）または移動情報リスト（D-013）、申請書テンプレート（D-008）
- 副作用（更新有無）：あり（Excelファイルの生成）

#### 3.2.2 入力（概念）

| 入力項目 | 説明 | 必須/任意 |
|---------|------|-----------|
| 申請種別 | 経費精算 / 交通費精算 | 必須 |
| 申請者情報 | 申請者名・申請日・所属部門等 | 必須 |
| 経費明細情報（経費精算時） | 購入日・金額・店舗名・品目・経費区分・業務目的等 | 条件付必須 |
| 移動情報リスト（交通費精算時） | 全区間の移動日・出発地・目的地・交通手段・費用・業務目的の一覧 | 条件付必須 |
| 申請書テンプレート | Excel形式の申請書テンプレートファイル（申請種別に対応） | 必須 |

> 入力パラメータの詳細（型・バリデーションルール・キー名）はツール基本設計・詳細設計フェーズで定義する

#### 3.2.3 出力（概念）

| 出力項目 | 説明 |
|---------|------|
| 申請書ファイル | 申請情報が書き込まれたExcel形式の申請書（下書き）。ファイル名：`{申請者名}_{申請書名}_{タイムスタンプ}.xlsx` |
| 生成結果 | 生成成功/失敗の結果 |

> 出力スキーマの詳細（キー名・型・構造）はツール詳細設計フェーズで定義する

#### 3.2.4 失敗時動作

| 失敗パターン | 動作 |

| Excel生成失敗 | エラーを返却。呼び出し元エージェントが再試行またはエスカレーション案内を提示する |

#### 3.2.5 承認・権限

| 権限ID | なし |
| 自律度 | Lv4（実行＋承認依頼） |
| 呼び出し可能エージェント | AG-002, AG-003 |

## 2. インタフェース詳細定義（要件定義レベル）

### 2.1 IF-ID：IF-001
### API名：Bedrock InvokeModel（会話・判断）

---

## 2.1.1 基本情報（👤）

| 項目 | 内容 |
|---|---|
| 外部システムID | EXT-001 |
| API提供元 | AWS Amazon Bedrock |
| 機能概要 | Claude Sonnet 4.5によるLLM推論。エージェントの会話・判断・生成処理を実行する |
| 利用目的 | 自然言語理解・申請種別判断・情報収集対話・申請ルール照会回答・チェック判断・申請書生成支援 |
| 想定利用エージェント | AG-001, AG-002, AG-003 |

---

## 2.1.2 I/F方式・非機能（👤）

| 項目 | 内容 |
|---|---|
| プロトコル | HTTPS |
| データ形式 | JSON |
| 認証方式 | 要件上未定義（AWS IAM想定） |
| レート制限 | 要件上未定義 |
| SLA | 要件上未定義 |
| 冪等性 | 不要（LLM推論は非冪等） |

---

## 2.1.3 リクエスト/レスポンス（概略）

### リクエスト項目（要件定義レベル）
| 項目名 | 必須 | 説明 |
|---|---|---|
| model_id | ○ | 使用モデルID（claude-sonnet-4-5相当） |
| messages | ○ | 会話履歴（ユーザー・アシスタントの交互メッセージ） |
| system | — | システムプロンプト（エージェント役割定義） |
| max_tokens | — | 最大トークン数 |

### レスポンス項目（要件定義レベル）
| 項目名 | 説明 |
|---|---|
| content | 生成テキスト（判断結果・回答・申請書内容） |
| stop_reason | 終了理由（end_turn / max_tokens） |
| usage | 入出力トークン数 |

※ 詳細スキーマは基本設計／詳細設計へ

---

## 2.1.6 エージェントツール化の可否評価（要件定義用）

| 観点 | 評価 |
|---|---|
| 自動化適性 | 高（Strands SDK経由で自動呼び出し） |
| 副作用リスク | 低（推論のみ、外部状態を変更しない） |
| 承認必須度 | 低（推論処理のみ） |
| ツール化可否 | 可（Strands SDKのエージェント基盤として組み込み） |

### API名：監査ログ書き込み（LOG-STD / LOG-HI）

---

## 2.2.1 基本情報（👤）

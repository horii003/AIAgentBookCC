# ファイルストア一覧

> **参照元（業務要件定義資料）:**
> - 業務データ定義.md（データ項目・機密区分）
> - 業務一覧.md（関連業務の特定）
> - 業務プロセス定義.md（データの更新頻度・CRUD主体の特定）
> - ユースケース定義.md（データの利用シーン）

> **スコープ定義:**
> 本システム（EAAS）はRDBを使用しない。永続化が必要なデータはすべてファイルストアにファイルベースで管理する。監査ログはJSONL（追記型）、運賃データはJSON、セッション状態はJSONで保存する。申請書はExcelファイルとして出力する（詳細はアウトプット一覧参照）。ナレッジ（社内申請ルール等）はファイルストア上のテキスト/JSONファイルを全文参照する。

## 2. ファイル詳細定義

### 2.1 ファイル：audit_log_std.jsonl（標準監査ログ）

- 概要：全エージェントの操作・判断をJSONL形式（1操作1行）で追記記録する
- 形式：JSONL（各行がJSON、改行区切り）
- 保存先：ファイルストア（EXT-003）
- 保持期間：要件上未定義
- 機密区分：D-2（機微）

#### 主要フィールド（概念定義）
| No | フィールド名 | 論理名 | 型 | 必須 | 代表値/例 | 備考 |
|---:|---|---|---|---|---|---|
| 1 | log_id | ログID | string（UUID） | ○ | — | 一意識別子 |
| 2 | log_type | ログ種別 | string | ○ | LOG-STD | 固定値 |
| 3 | request_id | リクエストID | string | ○ | — | セッション・リクエスト追跡 |
| 4 | actor | 実行者（AgentID） | string | ○ | AG-001, AG-002, AG-003 | |
| 5 | action | 実行アクション | string | ○ | ACT-READ-01等 | アクション分類ID |
| 6 | result | 実行結果 | string | ○ | success, error | |
| 7 | timestamp | 実行日時 | string（ISO8601） | ○ | 2026-05-01T12:00:00+09:00 | |

### 2.3 ファイル：fare_routes.json（運賃データ・電車経路）

- 概要：電車の経路テーブル。出発地・目的地→運賃の対応をJSON配列で定義する
- 形式：JSON
- 保存先：ファイルストア（EXT-003）
- 機密区分：D-1（社内）

#### フィールド構造（概念定義）
```json
{
  "routes": [
    {"from": "渋谷", "to": "品川", "fare": 210},
    {"from": "品川", "to": "渋谷", "fare": 210}
  ]
}
```

| フィールド名 | 論理名 | 型 | 備考 |
|---|---|---|---|
| routes | 経路リスト | 配列 | |
| routes[].from | 出発地（正規化済み駅名） | string | 「駅」接尾辞を除去した正規化済み駅名（BRL-18） |
| routes[].to | 目的地（正規化済み駅名） | string | 同上 |
| routes[].fare | 運賃（円） | integer | 円単位 |

### 2.5 ファイル：session_{id}.json（セッション状態）

- 概要：エージェント間の会話コンテキスト・収集済み申請情報を保持するセッションファイル。セッション終了後に削除する
- 形式：JSON
- 保存先：ファイルストア（EXT-003）
- 機密区分：D-2（機微）
- 削除タイミング：セッション終了後（詳細は実装フェーズで定義）

#### 主要フィールド（概念定義）
| No | フィールド名 | 論理名 | 型 | 備考 |
|---:|---|---|---|---|
| 1 | session_id | セッションID | string（UUID） | ファイル名に含む |
| 2 | applicant_name | 申請者名 | string | セッション開始時にAG-001が収集 |
| 3 | application_type | 申請種別 | string | 経費精算 / 交通費精算 |
| 4 | collected_info | 収集済み申請情報 | JSON | 申請書作成に必要な項目 |
| 5 | conversation_turn | 現在の対話回数 | integer | 上限30回（GRD-018） |
| 6 | approval_status | APR-001承認状態 | string | pending / approved / cancelled |
